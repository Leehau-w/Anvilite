## 一、产品是什么

### 1.1 一句话

**Anvilite（锻石）** — RPG游戏化桌面任务管理应用，角色是用户真实成长的镜像。

### 1.2 核心公式

```
认真的任务管理 × 角色即镜像 × 温和正向引导 × 完整成长记录
```

这四个要素缺一个都不是Anvilite：
- **认真的任务管理**：不是披着游戏皮的玩具，是真正好用的任务工具
- **角色即镜像**：角色状态反映用户真实状态，不是脱离现实的游戏角色
- **温和正向引导**：绝不惩罚，只有正向反馈和温和的状态表达
- **完整成长记录**：每一次努力都被记录，形成用户的"人生存档"

### 1.3 目标用户

有自我提升需求的年轻人（18-35岁），想要管理任务和习惯但觉得传统工具太枯燥，对RPG/游戏化有天然好感。核心痛点是：**想坚持但缺少正反馈，想记录但觉得繁琐**。

### 1.4 竞品差异

| 产品 | 定位 | Anvilite的差异 |
|------|------|---------------|
| Habitica | 游戏化习惯追踪，重游戏轻管理 | Anvilite任务管理更认真，角色是镜像不是宠物 |
| Forest | 专注计时，种树激励 | Anvilite是完整的任务+习惯管理系统，不只是计时 |
| Todoist | 纯任务管理，无游戏化 | Anvilite加入了成长体系、世界地图、角色养成 |
| Notion | 全能工具，学习成本高 | Anvilite开箱即用，专注于"完成事情并获得成长反馈" |

**Anvilite的独特价值**：把"做完一件事"的满足感可视化、可积累、可回顾。不是让用户"玩游戏"，而是让用户"看到自己的成长"。

## 二、产品设计红线

以下6条是产品设计的绝对底线，任何功能设计都不能违反：

| # | 红线 | 为什么 |
|---|------|--------|
| 1 | **不做任何惩罚机制** | 惩罚会让用户逃避使用产品。没有HP损失、角色死亡、连击惩罚。中断只是"失去加成"，不是"受到惩罚"。 |
| 2 | **角色=真实镜像** | 角色状态必须反映用户的真实行为（在学习→📚学习中），不能是脱离现实的游戏壳。 |
| 3 | **全量记录成长** | 每一次完成、升级、徽章都永久记录。用户的努力不会消失。GrowthEvent不自动清理。 |
| 4 | **矿石不收回** | 矿石代表"努力的物质证明"。即使撤销任务（XP会收回），矿石保留——因为努力本身真实发生过。 |
| 5 | **等级允许降级但不强调** | 撤销/删除任务导致XP不足时等级降低，但不做负面动画/提示（静默降级）。 |
| 6 | **中断用温和状态表达** | 用户长时间不用不叫"离线"或"缺席"，叫"远行中🌄"或"休憩中💤"。回来不叫"回归战场"，叫"归来🌅"。 |

## 三、功能模块全景

### 3.1 用户核心循环

```
完成任务/习惯
      ↓
获得 XP + 矿石 → 角色升级 → 解锁新称号
      ↓
区域繁荣提升 → 建筑从荒芜变辉煌 → 解锁装饰商店商品
      ↓
成就徽章 → 里程碑殿堂铭刻
      ↓
时光卷轴记录一切 → 回顾成长 → 淬火重铸（Prestige）
```

用户每天打开Anvilite的体验是：
1. 看到仪表盘问候「早上好，[角色名]」+ 今天该做什么
2. 完成任务 → 卡片飞出 → XP飘字 → 经验条增长 → （偶尔升级庆祝）
3. 进入世界地图看到自己的区域在逐渐繁荣
4. 翻看里程碑殿堂的徽章和碑石，回顾成长轨迹

### 3.2 导航结构（v0.3后）

```
侧边栏5个Tab：
  🏠 仪表盘（默认首页）
  📝 任务
  🗺️ 世界地图
  🏛️ 里程碑殿堂
  📋 SOP
```

### 3.3 各模块的产品定位

#### 仪表盘 — "今天我该做什么"

用户打开app的第一印象。回答核心问题：该做什么、做得怎么样了。

- 卡片式自由布局（用户可拖拽调整）
- 6张卡片：任务列表（简化版）、习惯（未完成的）、今日统计、角色迷你卡、快速创建、成长趋势热力图
- 时段问候语（早上好/下午好/晚上好 + 角色名）
- 热力图：月视图按天、周视图按4时段（1-6/6-12/12-18/18-24）

#### 任务系统 — "管理要做的事"

产品的功能地基。两种事物类型完全分离：

**一次性任务**（任务Tab）：
- 创建：标题必填，难度(★-★★★★★)默认3星，优先级(紧急/高/中/低)默认中，截止日期默认明天
- 子任务：轻量checklist模式（v0.3重构），只有标题+勾选框，最多3层嵌套，不获得XP
- 完成：勾选→卡片飞出→XP飘字→经验条增长。4-5星需二次确认
- 撤销：XP收回（可能降级），矿石不收回

**周期性习惯**（区域内部空间管理）：
- 8种重复模式（每天/工作日/每周严格或灵活/隔周/每月/每月固定日期/自定义）
- 连续性：完成+1、跳过-10%、缺席减半（有容错则抵消）
- 容错机制：首次连续7次得1个，之后每14次得1个，最多持有1个

#### 世界地图 — "看到成长的全景"

v0.3改为结构化卡片网格布局（像素风视觉保留，排列方式从自由散布改为整齐卡片）。

- 默认6个区域：家园/竞技场/书阁高塔/灵感工坊/锻造坊/档案馆
- 隐藏模板6个：泉水/议事厅/远征大厅/观测站/植物园/广场
- 总上限12个，用户可创建空白自定义区域
- 每个区域6级繁荣度：荒芜→聚落→丰饶→繁荣→鼎盛→辉煌
- 繁荣度由该区域累计XP驱动（对数增长）
- 每个区域有独特的6级视觉变化描述（辉煌级全部融入锻石元素）
- 点击卡片进入区域内部空间

#### 区域内部空间 — "在某个领域深耕"

进入区域后的沉浸式场景。像素场景全屏背景+半透明磨砂面板。

- 左侧：该区域的任务列表+习惯列表（习惯的主要管理入口在这里）
- 右侧：技能等级、统计数据、繁荣度进度、迷你热力图、装饰商店
- 角色在场景中：有doing任务→工作状态（各区域独特动作），无任务→空闲idle
- 装饰商店：用矿石购买区域装饰，按繁荣度等级解锁

#### 档案馆 — "数据复盘中心"

世界地图上的特殊固定区域（原"里程碑殿堂"位置，已更名）。

两个Tab：
- **数据总览**：5维统计（紧急程度分布、难度分布、区域时长排行、习惯坚持、高频行为），支持今日/本周/本月切换
- **时光卷轴**：横向时间轴，按月划分，默认展示Top3事件+分类计数，悬停放大查看详情，点击锁定展开

#### 里程碑殿堂 — "荣誉殿堂"

侧边栏独立Tab（第4个）。专属展厅式三栏布局：

- 左栏：角色大图+能力雷达图(动态维度最多12)+属性面板(等级/称号/XP/矿石)+称号展示柜
- 中栏：里程碑之碑（碑石装饰图+系统/自定义里程碑时间线）
- 右栏：成长统计总览(2×4网格)+成就徽章墙(8类28+个)

#### SOP模块 — "个人流程知识库"（v0.3新增）

侧边栏第5个Tab。核心价值是记录和查阅流程，附加能力是一键转为任务。

- 4种类型：日程型（时间轴）、流程型（有序步骤）、检查型（操作清单+执行模式）、清单型（无序勾选）
- 目录树+内容区布局（类似笔记软件）
- 用户自定义文件夹组织SOP
- 一键转为任务：整体转或勾选部分步骤转，生成父任务+checklist子任务
- 8个系统预设模板（工作日日程/周复盘/出差清单等）

#### 反馈动画 — "每一次完成都有仪式感"

5个核心动画精确定义参数：
1. 卡片飞出（~950ms）：勾选→scale→向右飞出→列表收拢
2. XP飘字（800ms）：向上弹出飘动淡出
3. 矿石飘字（800ms，延迟100ms）：偏右偏低，与XP形成层次
4. 经验条填充（600ms）：流畅增长+发光
5. 升级庆祝（~2200ms）：全屏遮罩+弹性数字+粒子+跨称号特殊文字

完整时序：0ms勾选 → 250ms飞出 → 300ms XP飘字 → 400ms矿石飘字 → 600ms经验条 → 1200ms结束（或触发升级庆祝到3400ms）

## 四、XP与经济系统

### 4.1 设计哲学

基础XP很小（1-8），让用户对每一点XP都有感知。加成比较大方（最高可叠加到+120%），奖励长期坚持和高难度挑战。矿石与XP 1:1获得但用途不同——XP用于升级，矿石用于消费（装饰+主题）。

### 4.2 数值表

**基础XP**（难度→XP）：★=1, ★★=2, ★★★=3, ★★★★=5, ★★★★★=8

**加成**（可叠加）：
| 加成 | 条件 | 比率 |
|------|------|------|
| 按时完成 | 有截止日期且按时完成 | +20% |
| 连续活跃 | 阶梯式：3天+10% / 7天+20% / 14天+30% / 30天+50% | 最高+50% |
| 高难度 | 4-5星任务 | +50% |

**频率折算**（防止高频习惯数值膨胀）：每日/工作日习惯×0.5，其余×1.0

**场景对照**（帮助理解难度定义）：
- ★(1)：整理桌面、倒垃圾
- ★★(2)：背20个单词、做10分钟拉伸
- ★★★(3)：写读书笔记、跑步30分钟
- ★★★★(5)：写一篇文章、准备一次演讲
- ★★★★★(8)：通过一门考试、跑完半程马拉松

### 4.3 等级节奏

公式：`round(5 × ln(level+1) × level)`

| 时间节点 | 预计等级 |
|---------|---------|
| 第1周 | Lv.4-5 |
| 第1月 | Lv.9-10 |
| 第3月 | Lv.15-16 |
| 第6月 | Lv.20-22 |
| 第1年 | Lv.28-32 |
| 第3年 | Lv.45-50 |

约束：第一年每2周至少升1级，后续每月至少升1级。不封顶。

### 4.4 称号体系

两套预设（8级）+ 用户自定义：

**锻造主题（默认）**：破壳(1-5) → 熔炼(6-10) → 锤炼(11-15) → 锋芒(16-20) → 极意(21-30) → 铸魂(31-40) → 锻石(41-50) → 不朽(51+)

**经典RPG**：新手 → 学徒 → 冒险者 → 游侠 → 英雄 → 大师 → 史诗 → 传奇

### 4.5 淬火重铸（Prestige）

Lv.51后解锁。等级归1，保留全部资产（矿石/装饰/主题/繁荣度）。获得转生星标🌟、动态流光称号、神话繁荣第7级、起源徽章。纯视觉特权，无数值膨胀。

## 六、12个区域的世界观

这是Anvilite最有品牌感的设计——每个区域从荒芜到辉煌都有独特的叙事，辉煌级全部融入"锻石"元素。

### 默认区域

| 区域 | 荒芜 | 辉煌 |
|------|------|------|
| **家园**（生活+日常） | 满地碎石与杂草的荒地 | 奇迹城堡，墙体闪烁锻石微光 |
| **竞技场**（运动/健身） | 杂草丛生的废弃泥地 | 战神巨像，永不熄灭的战意化作金色光芒 |
| **书阁高塔**（学习/阅读） | 爬满枯藤的残破石塔 | 塔顶星象仪汇聚智慧星图，化作耀眼丰碑 |
| **灵感工坊**（兴趣/创造） | 满是灰尘的废弃木屋 | 空间脱离物理定律，灵感如繁星环绕造物台 |
| **锻造坊**（工作/效率） | 冰冷残破火炉，红锈蛛网 | 炉火蜕变为锻石幽光，每次落锤震荡现实 |
| **档案馆**（数据统计） | 幽暗空旷大厅 | 不朽英灵殿，所有成就凝聚成悬浮巨型锻石 |

### 自定义模板区域

| 区域 | 适合场景 | 辉煌级 |
|------|---------|--------|
| 泉水 | 恢复/冥想 | 泉眼凝结液态锻石，永不枯竭的治愈光晕 |
| 议事厅 | 目标规划/复盘 | 桌面化作全息星际投影，意志如法则闪耀 |
| 远征大厅 | 中长期大项目 | 闪耀传送门洞开，璀璨光芒指引终极旅途 |
| 观测站 | 数据追踪/反思 | 化作全知之眼，星辰与锻石能量交织 |
| 植物园 | 情绪打卡/耐心培养 | 生命之树连接天地，枝叶间结出锻石果实 |
| 广场 | 碎片化习惯 | 悬浮于空中，沐浴神圣流光，璀璨明珠 |

## 十、团队与协作方式

### 10.1 角色分工

| 角色 | 职责 |
|------|------|
| **创始人** | 产品愿景、最终决策权、像素素材制作、驱动Claude Code开发 |
| **PM（你）** | 产品设计决策（交互/流程/优先级）、需求文档维护、待决项推进 |
| **技术总监** | 技术方案审核、架构决策、代码质量把控 |
| **开发** | Claude Code（AI编程），由创始人驱动 |

### 10.2 决策流程

```
创始人提出想法/问题
      ↓
PM讨论并出设计方案（交互/流程/数据）
      ↓
技术总监审核技术可行性和实现方案
      ↓
修订后确认 → Claude Code执行开发
      ↓
创始人验收 → PM确认产品体验
```

### 10.3 PM日常工作

1. **维护PRD**：功能变更时更新PRD或产出Patch文档
2. **做设计决策**：当创始人或开发遇到产品问题时给出方案
3. **推进待决项**：确保阻塞项在截止前解决（如过渡动画方式、hover交互设计）
4. **优先级排序**：当需求冲突时判断先做什么
5. **体验把控**：验收开发成果，确保符合设计规格

### 10.4 你的第一周

1. **读PRD + PRD Patch 2.0**（2-3小时）
2. **运行v0.2**：本地安装Electron应用，把所有功能点一遍
3. **读v0.3开发方案**（1小时）
4. **确认待决项**：
   - 世界地图过渡动画选哪种？（zoom推进/滑入/淡入淡出）
   - 任务hover交互怎么改？（需要出设计方案）
5. **与创始人对齐**：了解他当前最关心的产品问题

## 十一、常见问题

### Q：Anvilite和Habitica的核心差异是什么？
A：Habitica把自己当游戏（打怪升级掉血），Anvilite把自己当镜子（角色反映你的真实状态）。Habitica有惩罚（掉血扣金币），Anvilite没有任何惩罚。Habitica的任务管理很简单，Anvilite有认真的任务系统（子任务、优先级、截止日期、分类筛选）。

### Q：为什么子任务不获得XP？
A：防止用户把一个★★★任务拆成10个子任务来刷经验。子任务只是父任务的"步骤/检查项"，完成父任务才获得XP。这保证了XP的含金量。

### Q：为什么有矿石又有XP，不能合并吗？
A：XP用于升级（代表成长），矿石用于消费（代表购买力）。如果用XP买装饰，买完后XP减少→等级下降→用户会犹豫要不要花钱→消费体验很差。矿石解决了这个问题：花矿石不影响等级。

### Q：SOP和任务有什么关系？
A：SOP是"做事的方法论"（参考手册），任务是"要做的事"（执行清单）。SOP可以一键转为任务，但转化后两者独立——修改SOP不影响任务，修改任务不影响SOP。

### Q：世界地图为什么要从自由画布改成卡片？
A：创始人实际使用后觉得自由拖拽不美观、交互混乱。卡片网格更整齐、更好操作，同时保留了像素风视觉元素。这是"好看但不好用"到"好看又好用"的改进。

### Q：像素素材的事PM需要管吗？
A：不需要。素材由创始人自己制作（Aseprite+Midjourney），这不是产品功能开发，是独立的美术工作。PM只需要关注功能和交互层面。

### Q：后端什么时候做？
A：v0.3暂缓，v0.4启动。方案已设计好（Supabase全套），前端打磨完再做。PM可以开始思考云同步的用户体验（冲突提示、离线状态、首次同步流程等）。
