# Anvilite · 锻石 — 反馈动画系统规格文档

> **版本**: v1.0
> **日期**: 2026-03-28
> **状态**: 已锁定

---

## 一、模块定位

反馈动画是产品质感的关键。每一个完成、升级、解锁的瞬间都需要恰到好处的视觉反馈——不过度夸张制造疲劳，也不过于克制丧失成就感。

### 1.1 动画分类

| 类别 | 动画 | 频率 | 定义精度 |
|------|------|------|---------|
| 核心 | 任务卡片飞出 | 每次完成任务 | 精确参数 |
| 核心 | XP飘字 | 每次完成任务/习惯 | 精确参数 |
| 核心 | 矿石飘字 | 每次完成任务/习惯 | 精确参数 |
| 核心 | 经验条填充 | 每次获得XP | 精确参数 |
| 核心 | 升级庆祝 | 升级时（约每1-2周） | 精确参数 |
| 次要 | 习惯连续里程碑 | 达到特定连续次数 | 方向定义 |
| 次要 | 徽章解锁 | 获得徽章时 | 方向定义 |
| 次要 | 区域繁荣升级 | 区域升级时 | 方向定义 |
| 次要 | 角色状态切换 | 状态变化时 | 方向定义 |

### 1.2 全局动画原则

- **缓动函数**：默认使用 `cubic-bezier(0.4, 0, 0.2, 1)`（Material Design ease-in-out），除非特别指定
- **可中断**：所有动画可被用户下一个操作中断（不阻塞交互）
- **可叠加**：多个动画可同时播放（如XP飘字+经验条填充+卡片飞出）
- **跟随主题**：所有颜色使用CSS变量，跟随主题切换
- **减弱动画模式**：如果系统开启了「减少动画」辅助功能，所有动画时长减半或直接跳过

## 二、核心动画 — 精确参数

### 2.1 任务卡片飞出

**触发**：任务完成（勾选框点击 + 4-5星确认后）

**动画流程**：

```
阶段1：勾选反馈（100ms）
  - 勾选框从空心变实心（success色）
  - 内部出现 ✓ 标记
  - 整个卡片轻微缩放 scale(1.02)

阶段2：准备飞出（150ms 延迟）
  - 卡片保持 scale(1.02)
  - 用户能看到 ✓ 标记的满足感

阶段3：飞出（400ms）
  - translateX: 0 → 120%（向右飞出屏幕外）
  - opacity: 1 → 0（最后100ms淡出）
  - rotate: 0 → 3deg（轻微旋转增加动感）
  - 缓动: cubic-bezier(0.4, 0, 1, 1)（ease-in，越来越快）

阶段4：列表收拢（300ms）
  - 飞出后留下的空隙，上下卡片平滑合拢
  - 缓动: cubic-bezier(0.4, 0, 0.2, 1)
```

**总时长**：100 + 150 + 400 + 300 = ~950ms

**CSS参考**：
```css
@keyframes card-fly-out {
  0% { transform: translateX(0) rotate(0deg) scale(1.02); opacity: 1; }
  75% { transform: translateX(100%) rotate(2deg); opacity: 1; }
  100% { transform: translateX(120%) rotate(3deg); opacity: 0; }
}
.card-fly-out {
  animation: card-fly-out 400ms cubic-bezier(0.4, 0, 1, 1) forwards;
  animation-delay: 250ms; /* 勾选反馈 + 延迟 */
}
```

### 2.2 XP飘字

**触发**：任务/习惯完成后，紧接卡片飞出（或勾选完成）

**起始位置**：完成的任务卡片原来的位置（卡片飞走后从该位置飘出）

**动画参数**：

```
内容: "+X XP"
字体: Inter Semi Bold (600), 16px
颜色: var(--color-xp)，即XP色
起始位置: 任务卡片中心偏右
结束位置: 向上偏移 60px

动画:
  时长: 800ms
  translateY: 0 → -60px
  opacity: 0 → 1 → 1 → 0
    - 0-200ms: 淡入 (0→1)
    - 200-600ms: 保持显示 (1)
    - 600-800ms: 淡出 (1→0)
  scale: 0.8 → 1.1 → 1.0
    - 0-200ms: 放大弹出 (0.8→1.1)
    - 200-400ms: 回弹 (1.1→1.0)
    - 400-800ms: 保持 (1.0)
  缓动(位移): cubic-bezier(0, 0, 0.2, 1)（减速上升）
```

**CSS参考**：
```css
@keyframes xp-float {
  0% { transform: translateY(0) scale(0.8); opacity: 0; }
  25% { transform: translateY(-15px) scale(1.1); opacity: 1; }
  50% { transform: translateY(-30px) scale(1.0); opacity: 1; }
  75% { transform: translateY(-48px); opacity: 1; }
  100% { transform: translateY(-60px); opacity: 0; }
}
```

### 2.3 矿石飘字

**触发**：与XP飘字同时，但位置偏移避免重叠

**动画参数**：

```
内容: "+X ⛏"
字体: Inter Semi Bold (600), 14px（比XP飘字小一号）
颜色: var(--color-secondary)，即矿石色
起始位置: XP飘字右侧偏移 40px
结束位置: 向上偏移 50px（比XP少，形成高低差）

动画:
  时长: 800ms
  延迟: 100ms（比XP飘字晚出现一点，形成先后层次）
  translateY: 0 → -50px
  opacity: 与XP飘字相同的淡入-保持-淡出节奏
  scale: 0.8 → 1.05 → 1.0（比XP弹跳幅度小）
  缓动: 与XP飘字相同
```

**XP和矿石飘字的相对位置**：
```
          +5 XP     ← 先出现，飘更高
            +5 ⛏   ← 100ms后出现，偏右偏低
   ─────────────────
   [已飞出的任务卡片位置]
```

### 2.4 经验条填充

**触发**：XP飘字出现后300ms开始（让用户先看到飘字数字）

**位置**：底部状态栏的经验条

**动画参数**：

```
动画:
  时长: 600ms
  width: oldPercent% → newPercent%（百分比变化）
  缓动: cubic-bezier(0.4, 0, 0.2, 1)（平滑减速）
  
附加效果:
  填充过程中经验条发出轻微光晕（box-shadow: 0 0 8px var(--color-accent)）
  填充完成后光晕消退（200ms淡出）

数字更新:
  经验条右侧的 "当前/所需" 数字在填充过程中实时递增
  数字递增使用 requestAnimationFrame 逐帧更新
  递增速度与进度条同步
```

**如果触发升级**：经验条先填满到100%，然后触发升级庆祝动画（见2.5），经验条重置为新等级的起始值。

```
经验条填充到100% (600ms)
  → 短暂停留在满状态 (200ms)
  → 触发升级庆祝动画
  → 经验条闪烁一次后重置为新百分比 (300ms)
```

### 2.5 升级庆祝动画

**触发**：经验条填满触发升级

**动画流程**：

```
阶段1：全屏遮罩 (200ms)
  - 半透明黑色遮罩从透明淡入（opacity 0 → 0.5）
  - 背景内容轻微模糊（backdrop-filter: blur(4px)）

阶段2：等级数字弹出 (500ms)
  - 屏幕中央出现大号等级数字
  - 内容: "Lv.X → Lv.Y" 或跨称号时 "称号名"
  - 字号: 48px（等级数字）/ 24px（称号名）
  - 颜色: accent色（等级）/ xp色（称号）
  - scale: 0 → 1.2 → 1.0（弹性弹出）
  - 缓动: cubic-bezier(0.34, 1.56, 0.64, 1)（弹簧效果）

阶段3：粒子效果 (1000ms)
  - 从等级数字中心向四周发射粒子
  - 粒子数量: 20-30个
  - 粒子颜色: accent色 + xp色 混合
  - 粒子轨迹: 从中心向外扩散，带重力弧线
  - 粒子大小: 3-6px圆形
  - 粒子逐渐淡出

阶段4：消退 (500ms)
  - 等级数字向上飘移并淡出
  - 遮罩淡出
  - 粒子消散
  - 底部状态栏的等级数字更新（闪烁一次accent色）

跨称号时额外效果:
  - 阶段2中先显示等级变化，然后称号文字从下方滑入
  - 称号文字: H2字号(20px), accent色, fontWeight 700
  - 显示: "你已从「熔炼」晋升为「锤炼」！"
  - 停留 1500ms 让用户阅读
```

**总时长**：约 2200ms（普通升级）/ 3500ms（跨称号升级）

**可跳过**：用户点击任意位置或按ESC可立即结束动画（直接跳到阶段4的结果状态）

## 三、次要动画 — 方向定义

以下动画只定义方向和大致感觉，具体参数留给开发时调整。

### 3.1 习惯连续里程碑反馈

**触发**：习惯连续完成次数达到 7/14/21/30/50/100... 等节点

**方向**：
- 在习惯卡片位置弹出连续次数的大号数字（如 🔥30）
- 数字从小到大弹出，配合火焰emoji的放大效果
- 比普通XP飘字更醒目，但不如升级庆祝那么隆重
- 持续约1000-1500ms
- 不阻塞交互，不需要遮罩

**参考感觉**：类似游戏中"Combo x30!"的反馈

### 3.2 徽章解锁动画

**触发**：获得新徽章

**方向**：
- 屏幕右上角弹出徽章通知卡片（从右侧滑入）
- 卡片中显示徽章图标（从灰色变为彩色）+ 徽章名称
- 徽章图标有轻微发光/闪烁效果
- 通知卡片停留3秒后自动滑出
- 用户可点击卡片跳转到里程碑殿堂查看
- 不阻塞交互

**参考感觉**：类似Steam成就解锁通知

### 3.3 区域繁荣度升级动画

**触发**：区域繁荣度达到新等级

**方向**（世界地图视角）：
- 该区域建筑发出光芒（从旧形态渐变到新形态）
- 建筑变化过渡约500-800ms
- 变化完成后区域周围短暂发光（光晕扩散后消散）
- 发光效果约1000ms
- 底部Toast通知：「[区域名] 升级为 [繁荣等级]！」

**方向**（如果用户在该区域内部）：
- 场景背景发生变化（旧场景淡出新场景淡入）
- 过渡约800ms
- 角色短暂做出庆祝动作

### 3.4 角色状态切换动画

**触发**：角色状态变化（如从充电中→活跃、活跃→工作中等）

**方向**：
- 角色sprite做过渡动画（旧状态淡出，短暂的过渡姿态，新状态淡入）
- 过渡总时长约500-800ms
- 状态文字同步更新（淡入淡出）
- 如果是从非活跃状态→活跃/工作（回归）：
  - 角色先显示returning状态（带旅行痕迹）约1-2秒
  - 然后过渡到活跃/工作状态
- 不需要特殊特效，自然平滑即可

## 四、Toast通知动画

所有Toast通知使用统一的动画参数（已在设计规范中定义，此处汇总）：

```
入场: 从下方滑入 + 淡入
  translateY: 20px → 0
  opacity: 0 → 1
  时长: 300ms
  缓动: cubic-bezier(0.4, 0, 0.2, 1)

停留: 5秒

退场: 淡出
  opacity: 1 → 0
  时长: 200ms

位置: 屏幕底部居中，距底部24px
```

## 五、任务完成的完整动画时间线

将所有核心动画串联，展示一次任务完成的完整反馈流程：

```
时间轴 (毫秒):

0ms     ─ 勾选反馈开始（✓ 出现，卡片 scale 1.02）
100ms   ─ 勾选反馈完成
250ms   ─ 卡片开始飞出（向右 + 旋转 + 淡出）
300ms   ─ XP飘字开始（"+X XP" 向上弹出）
400ms   ─ 矿石飘字开始（"+X ⛏" 偏右偏低弹出）
550ms   ─ Toast出现（"已完成 · 撤销"）
600ms   ─ 经验条开始填充
650ms   ─ 卡片飞出完成
650ms   ─ 列表空隙开始收拢
950ms   ─ 列表收拢完成
1100ms  ─ XP/矿石飘字淡出
1200ms  ─ 经验条填充完成

如果触发升级：
1200ms  ─ 经验条满
1400ms  ─ 升级遮罩淡入
1600ms  ─ 等级数字弹出 + 粒子发射
2600ms  ─ 粒子消散开始
3100ms  ─ 等级数字淡出，遮罩消退
3400ms  ─ 状态栏等级更新，经验条重置

5550ms  ─ Toast消失（5秒后）
```

## 六、世界地图进入/退出动画（汇总）

已在世界地图模块中定义，此处汇总参数：

### 6.1 进入区域（zoom推进）

```
触发: 点击信息面板「进入区域」按钮
流程:
  1. 信息面板收起 (200ms, ease-in)
  2. 其他区域淡出 (200ms, 与zoom并行)
  3. 镜头向目标区域zoom推进 (800ms)
     - scale: 1 → 10（视觉上放大）
     - 镜头中心对准目标区域
     - 缓动: cubic-bezier(0.4, 0, 0.2, 1)
  4. 最后200ms: 区域建筑淡出，内部空间淡入（交叉淡入淡出）
总时长: ~1000ms
```

### 6.2 退出区域（zoom拉远）

```
触发: 「← 返回地图」按钮 或 ESC键
流程: 进入动画反向播放
  1. 内部空间淡出，区域建筑淡入 (200ms)
  2. 镜头zoom拉远 (800ms)
  3. 其他区域淡入 (200ms)
总时长: ~1000ms
缓动: cubic-bezier(0.4, 0, 0.2, 1)
```

## 七、UI过渡动画（通用）

### 7.1 右侧抽屉

```
打开: 从右侧滑入
  translateX: 100% → 0
  时长: 250ms
  缓动: cubic-bezier(0.4, 0, 0.2, 1)
  遮罩同步淡入 (opacity 0 → 0.3)

关闭: 向右侧滑出
  translateX: 0 → 100%
  时长: 200ms
  缓动: cubic-bezier(0.4, 0, 1, 1)
  遮罩同步淡出
```

### 7.2 区域内部面板展开/收起

```
展开: 左右面板从两侧滑入
  左面板: translateX(-100%) → 0
  右面板: translateX(100%) → 0
  时长: 300ms
  缓动: cubic-bezier(0.4, 0, 0.2, 1)

收起: 反向滑出
  时长: 250ms
```

### 7.3 底部信息栏（世界地图）

```
滑入: translateY(100%) → 0
时长: 250ms
缓动: cubic-bezier(0.4, 0, 0.2, 1)

滑出: translateY(0) → 100%
时长: 200ms
```

### 7.4 模态弹窗

```
打开:
  遮罩: opacity 0 → 0.5 (200ms)
  弹窗: scale(0.95) opacity(0) → scale(1) opacity(1) (250ms)
  缓动: cubic-bezier(0.4, 0, 0.2, 1)

关闭:
  弹窗: opacity(1) → opacity(0) (150ms)
  遮罩: opacity 0.5 → 0 (200ms)
```

### 7.5 卡片悬停

```
hover:
  阴影: sm → md (150ms)
  背景: 轻微变亮 (150ms)
  缓动: ease-out

unhover:
  阴影: md → sm (200ms)
  缓动: ease-in
```

### 7.6 新任务入场

```
新创建的任务卡片入场:
  translateY: -10px → 0
  opacity: 0 → 1
  时长: 250ms
  缓动: cubic-bezier(0.4, 0, 0.2, 1)
```

## 八、Framer Motion 实现提示

由于项目使用 Framer Motion 作为动画库，以下是关键动画的实现建议：

```typescript
// 任务卡片飞出
const flyOutVariants = {
  initial: { x: 0, rotate: 0, opacity: 1, scale: 1.02 },
  exit: {
    x: "120%",
    rotate: 3,
    opacity: 0,
    transition: {
      duration: 0.4,
      ease: [0.4, 0, 1, 1],
      opacity: { delay: 0.3, duration: 0.1 }
    }
  }
};

// XP飘字
const xpFloatVariants = {
  initial: { y: 0, scale: 0.8, opacity: 0 },
  animate: {
    y: -60,
    scale: [0.8, 1.1, 1.0, 1.0],
    opacity: [0, 1, 1, 0],
    transition: {
      duration: 0.8,
      ease: [0, 0, 0.2, 1],
      scale: { times: [0, 0.25, 0.5, 1] },
      opacity: { times: [0, 0.25, 0.75, 1] }
    }
  }
};

// 升级弹出
const levelUpVariants = {
  initial: { scale: 0, opacity: 0 },
  animate: {
    scale: 1,
    opacity: 1,
    transition: {
      duration: 0.5,
      ease: [0.34, 1.56, 0.64, 1] // 弹簧效果
    }
  }
};

// 列表项 AnimatePresence
// 使用 <AnimatePresence> 包裹任务列表
// 每个任务卡片使用 layout 属性实现列表重排动画
```

## 九、与其他模块的关系

| 触发源 | 动画 |
|--------|------|
| 任务完成 | 勾选→飞出→XP飘字→矿石飘字→经验条填充→(升级) |
| 习惯完成 | XP飘字→矿石飘字→经验条填充→(连续里程碑) |

| 获得徽章 | 徽章解锁通知 |
| 区域繁荣升级 | 区域发光变化 |
| 角色状态变化 | 状态过渡动画 |
| 世界地图进入/退出 | zoom推进/拉远 |
| 所有面板 | 滑入/滑出过渡 |
