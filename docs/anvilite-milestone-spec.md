# Anvilite · 锻石 — 里程碑殿堂模块规格文档

> **版本**: v1.0
> **日期**: 2026-03-28
> **状态**: 已锁定

---

## 一、模块定位

里程碑殿堂是世界地图上的一个**固定特殊区域**，不可删除不可改名。它是用户所有成就和成长历程的"荣誉殿堂"，包含角色属性、能力雷达图、成就徽章、自定义里程碑、成长统计总览和称号展示柜。

与其他区域内部空间不同，里程碑殿堂使用**专属的展厅式布局**，而非全屏场景+半透明面板。

---

## 二、繁荣度

里程碑殿堂的繁荣度由**角色总等级**驱动（而非某个技能维度）。

| 繁荣等级 | 角色等级范围 | 视觉描述 |
|---------|------------|---------|
| 荒芜 | Lv.0 | 幽暗空旷大厅，粗糙石墙，空无一物 |
| 聚落 | Lv.1-5 | 入口火把点亮，第一座粗糙石雕被郑重安放 |
| 丰饶 | Lv.6-15 | 两侧排列整齐铭牌，记录已跨越的征程 |
| 繁荣 | Lv.16-25 | 殿堂不断扩建，战利品与荣誉勋章熠熠生辉 |
| 鼎盛 | Lv.26-40 | 穹顶镶嵌彩色玻璃，神圣光束打在每座丰碑上 |
| 辉煌 | Lv.41+ | 不朽英灵殿，所有成就凝聚成一颗悬浮巨型锻石 |


### 3.2 布局规格

- **背景**：殿堂像素场景全屏（随繁荣度变化）
- **三栏布局**：左约30% / 中约30% / 右约30%，三栏均使用半透明磨砂面板
- **面板样式**：与区域内部空间一致（rgba 75% + blur 8px + border），但布局为展厅式三栏而非功能面板
- **中央碑石**：视觉上应有一个像素风的碑石/纪念碑图形元素作为装饰，嵌入在中央面板中
- **纵向可滚动**：每栏内容超出可见区域时独立滚动

### 3.3 进入/退出

- 从世界地图进入：与其他区域一致的zoom推进动画（800ms）
- 退出：左上角「← 返回地图」按钮 或 ESC键，反向zoom

## 四、左侧区域：角色属性

### 4.1 角色展示

- 角色大图（像素风，区别于仪表盘迷你卡，这里是完整展示）
- 角色下方显示当前状态（⚡ 就绪 / 📚 学习中 等）
- 角色图随繁荣度变化（与殿堂场景匹配）

### 4.2 能力雷达图

- **维度**：每个已激活区域一个维度（最多12个）
- **维度值**：该区域的技能等级（与繁荣度驱动用相同数据）
- **维度标签**：区域名称（如「书阁高塔」「竞技场」）
- **视觉**：
  - 蛛网式雷达图
  - 填充色：accent色 20%透明度
  - 边线：accent色
  - 各维度顶点有小圆点
  - 背景网格：border色
- **动态**：维度数随用户激活的区域变化（新增区域→新增一个维度）
- **里程碑殿堂不作为雷达图维度**（它是展示方，不是数据源）

### 4.3 属性面板

```
┌─────────────────────┐
│  Lv.12              │ ← H1字号，accent色，数字字体
│  锤炼 Hammering     │ ← H3字号，text色
│                     │
│  [████████░░] 45/208│ ← 经验条，accent色
│                     │
│  总经验    1,247 XP  │
│  总矿石    ⛏ 892    │
│  使用天数   86 天     │
└─────────────────────┘
```

- 等级数字：H1(28px)，fontWeight 800，accent色，数字字体规则
- 称号：H3(16px)，fontWeight 600，text色
- 经验条：高度6px（比底部状态栏的4px稍大），accent色填充
- 统计数字：数字字体规则

### 4.4 称号展示柜

### 5.3 里程碑卡片样式

**系统里程碑**：
- 左侧图标：对应徽章图标（🏆 等）
- 标题：系统生成的描述（如「连续活跃30天」）
- 日期：Small字号，dim色
- 不可编辑，不可删除

**用户自定义里程碑**：
- 左侧图标：⭐（或用户选择的图标）
- 标题：用户填写
- 日期：用户设定
- 描述：用户填写的感想（可选）
- 可编辑，可删除

### 5.4 创建自定义里程碑

**入口一：碑石列表底部的「+ 刻碑」按钮**

点击后弹出创建面板：
```
┌─────────────────────────┐
│  刻下里程碑               │
│                         │
│  标题 _________________ │ ← 必填
│  日期 [2026-03-28]      │ ← 默认今天
│  感想 _________________ │ ← 选填，多行
│       _________________│
│                         │
│  [取消]        [刻碑 ✓]  │
└─────────────────────────┘
```

**入口二：在时间线中标记**

- 在时间线页面中，任何GrowthEvent右侧有一个「⭐ 标记为里程碑」按钮
- 点击后该事件同时出现在里程碑殿堂的碑石列表中
- 可追加填写感想

### 5.5 里程碑与GrowthEvent的关系

- 系统里程碑：自动记录一条 type=milestone 的GrowthEvent，同时显示在碑石和时间线中
- 用户自定义里程碑（通过「刻碑」创建）：记录一条 type=custom_milestone 的GrowthEvent
- 用户自定义里程碑（通过时间线标记）：在原有GrowthEvent上添加 isMilestone=true 标记

## 八、徽章颁发流程

### 8.1 触发检查

每次以下事件发生时，检查是否有新徽章可颁发：
- 任务/习惯完成
- 等级提升
- 连续活跃天数更新
- 区域繁荣度升级
- 主题皮肤解锁

### 8.2 颁发反馈

1. 全局Toast通知：「🏆 获得徽章：[徽章名]」
2. 记录GrowthEvent（type: badge_earned）
3. 里程碑殿堂的碑石列表自动新增一条系统里程碑
4. 徽章墙中对应徽章从灰色变为彩色（发光动画）
5. 如果用户当前不在里程碑殿堂：下次进入时在对应徽章上显示「新」标记

### 8.3 徽章数据结构

```typescript
interface Badge {
  id: string;
  category: string;          // 系列名（起步/等级/坚持/区域/习惯/任务量/投入/收集）
  name: string;              // 徽章名
  description: string;       // 描述文字
  condition: string;         // 解锁条件描述
  icon: string;              // 图标标识
  earned: boolean;           // 是否已获得
  earnedAt: string | null;   // 获得时间
  isNew: boolean;            // 是否有「新」标记未查看
}
```

## 十、边界情况

### 10.1 新用户（无任何成就）

- 角色展示：Lv.1，破壳，雷达图只有默认5个维度且全部为0
- 碑石列表：空，显示「你的故事即将在这里开始」+ 「+ 刻碑」按钮
- 统计：全部为0
- 徽章墙：全部为灰色未解锁状态，显示解锁条件
- 称号展示柜：只有「破壳」已解锁

### 10.2 雷达图维度变化

- 用户新增区域：雷达图自动新增一个维度（从5角变6角等）
- 用户删除区域：雷达图减少一个维度
- 维度数最少5个（默认区域，不含里程碑殿堂），最多12个（全部区域激活，不含里程碑殿堂）
- 只有1-2个区域时不显示雷达图，改为柱状图

### 10.3 降级影响

- 如果用户因撤销/删除任务导致等级降级：
  - 等级和称号回退
  - 已获得的等级系列徽章**不收回**（徽章永久保留）
  - 碑石中的系统里程碑记录**不删除**

### 10.4 殿堂繁荣度

- 殿堂繁荣度由角色总等级驱动
- 角色降级可能导致殿堂繁荣度降级（因为它是实时反映等级的）
- 殿堂场景会相应变化

## 十一、与其他模块的关系

| 交互 | 说明 |
|------|------|
| 里程碑殿堂 → 世界地图 | 通过「返回地图」退出，zoom拉远动画 |
| 里程碑殿堂 → XP系统 | 等级/称号/XP数据来源 |
| 里程碑殿堂 → 矿石系统 | 矿石余额数据来源 |
