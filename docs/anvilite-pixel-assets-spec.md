# Anvilite · 锻石 — 像素素材方案规格文档

> **版本**: v1.0
> **日期**: 2026-03-28
> **状态**: 已锁定

---

## 一、方案概述

采用 **D+E融合方案**：

| 素材类型 | 制作方式 | 理由 |
|---------|---------|------|
| 世界地图区域建筑（外部视角） | 开源素材包为基础 + 修改扩展 | 12个区域需要风格统一，素材包保证一致性 |
| 区域内部空间场景 | AI生成为主 + 手动微调 | 各场景独立，不需要互相一致，AI自由度高 |
| 地图装饰元素 | 素材包提取 | 与建筑同一风格 |
| 角色sprite | 素材包基础 + 手动修改动画帧 | 角色在所有场景出现，必须与外部建筑风格一致 |
| 装饰物品 | AI生成 + 手动微调 | 每个区域独特，量大，适合AI批量生成 |
| 徽章图标 | AI生成或手绘 | 数量少（~28个），独立展示不需要与场景一致 |
| 空状态插画 | AI生成 | 独立展示 |
| 碑石（里程碑殿堂） | AI生成 + 手动微调 | 只有1个元素6级变化 |

## 三、风格规范

### 3.1 像素规格

| 参数 | 值 |
|------|-----|
| 像素密度 | 16×16 或 32×32 基础网格 |
| 渲染倍率 | 2x-4x（实际显示尺寸=基础尺寸×倍率） |
| 视角 | 等距（Isometric），约26.57°角（标准2:1等距） |
| 抗锯齿 | 不使用（保持像素锐利边缘） |
| 缩放方式 | nearest-neighbor（不模糊） |

### 3.2 调色板约束

所有素材统一使用一个**固定调色板**，确保风格一致：

- 基础调色板选择：从开源素材包中提取调色板，或使用 Lospec 上的经典像素画调色板
- 建议调色板大小：32-48色
- 包含：6级灰度、暖色系（建筑/木材）、冷色系（石头/金属）、自然色（草地/水）、强调色（火焰/魔法/锻石光芒）
- AI生成的素材必须在后期统一到此调色板

### 3.3 等距视角标准

```
        /\
       /  \
      /    \
     /      \
    /________\
    \        /
     \      /
      \    /
       \  /
        \/

标准等距比例：宽:高 = 2:1
地面砖块：64×32px（2x渲染时）
建筑高度：根据内容，但底面始终遵循2:1
```

### 3.4 光照方向

- 统一光源方向：**左上方**（所有素材一致）
- 阴影投向：右下方
- AI生成时在prompt中指定光照方向

## 四、外部建筑素材（素材包方案）

### 4.1 素材包选择标准

| 标准 | 要求 |
|------|------|
| 风格 | 等距像素风，中世纪/奇幻主题 |
| 包含内容 | 多种建筑类型（小屋/塔楼/城堡/工坊等） |
| 繁荣度兼容 | 同一建筑有多个等级变化（或可拆分组合实现） |
| 环境元素 | 包含树木/草地/水/路径等 |
| 授权 | MIT / CC0 / CC-BY（允许商用和修改） |
| 调色板 | 统一且可提取 |

### 4.2 推荐素材包来源

| 来源 | 说明 |
|------|------|
| itch.io | 最大的像素素材市场，搜索 "isometric pixel art pack" |
| Kenney.nl | 高质量免费CC0素材，有等距建筑系列 |
| OpenGameArt.org | 开源游戏素材库 |
| Pixel Frog | 高质量像素素材 |

### 4.3 繁荣度变化的实现策略

**不是每级一整张图，而是分层叠加**：

```
辉煌级 = 基础建筑(6级) + 环境层(6级) + 装饰层(6级) + 特效层(6级)

每层独立变化：
  基础建筑层：帐篷 → 木屋 → 石屋 → 庄园 → 城堡 → 奇迹城堡
  环境层：    荒地 → 草地 → 花园 → 大花园 → 魔法花园 → 终极花园
  装饰层：    无 → 篝火 → 路灯 → 旗帜 → 雕像 → 锻石光芒
  特效层：    无 → 无 → 轻微 → 烟囱 → 粒子 → 全屏光效
```

这样每级不需要画一整张新图，只需要替换/添加层级。

### 4.4 素材包不够时的补充策略

- 基础建筑如果素材包中没有完美匹配的：用最接近的素材在Aseprite中修改
- 高等级建筑（鼎盛/辉煌）素材包通常没有：用AI生成后统一调色板
- 锻石特效元素：手绘或AI生成

## 五、内部场景素材（AI生成方案）

### 5.1 AI工具选择

| 工具 | 优势 | 劣势 | 推荐用途 |
|------|------|------|---------|
| Midjourney | 质量最高，像素风理解好 | 付费，一致性控制有限 | 主要生成工具 |
| Stable Diffusion + LoRA | 可训练自定义风格，一致性可控 | 需要本地部署，学习成本高 | 进阶选择 |
| DALL-E 3 | 简单易用 | 像素风效果一般 | 备选 |
| Piskel / Aseprite | 手动像素编辑 | 慢，需要画功 | 后期微调 |

### 5.2 Prompt模板

为保证AI生成的素材风格统一，使用标准化prompt模板：

**内部场景prompt模板**：
```
isometric pixel art interior scene, [场景描述], 
[繁荣度描述], 32-bit pixel art style, 
warm lighting from upper left, fantasy RPG theme,
detailed pixelated textures, clean pixel edges,
no anti-aliasing, [调色板描述],
game asset, transparent background
```

**示例 — 书阁高塔·丰饶级内部**：
```
isometric pixel art interior scene, 
a cozy stone tower library with wooden bookshelves 
filled with scrolls and books, oil lamps glowing warmly, 
a spiral staircase in the corner, ink and quill on a desk,
32-bit pixel art style, warm lighting from upper left, 
fantasy RPG theme, detailed pixelated textures, 
clean pixel edges, no anti-aliasing, 
earth tones with warm amber highlights,
game asset, transparent background
```

### 5.3 AI生成后的处理流程

```
1. AI生成原始图 (Midjourney/SD)
      ↓
2. 筛选最佳结果
      ↓
3. 在Aseprite中打开
      ↓
4. 统一调色板（映射到项目固定调色板）
      ↓
5. 修正像素网格对齐（去除半像素/模糊）
      ↓
6. 调整等距透视角度（如果偏差）
      ↓
7. 统一光照方向（左上方）
      ↓
8. 裁剪/调整尺寸到标准规格
      ↓
9. 导出 PNG（透明背景）
```

### 5.4 一致性检查清单

每张AI生成的素材在纳入项目前，检查：

- [ ] 调色板是否统一到项目调色板
- [ ] 像素密度是否一致（不能有些精细有些粗糙）
- [ ] 等距透视角度是否正确（2:1比例）
- [ ] 光照方向是否为左上方
- [ ] 边缘是否锐利（无抗锯齿模糊）
- [ ] 整体氛围是否与该区域主题匹配
- [ ] 尺寸是否符合标准规格

## 六、角色Sprite

### 6.1 角色规格

| 参数 | 值 |
|------|-----|
| 基础尺寸 | 16×24px 或 24×32px（等距人物比例） |
| 渲染倍率 | 与场景一致（2x-4x） |
| 动画帧数 | idle: 4帧, 工作: 4-6帧, 状态: 2-4帧 |
| 帧率 | 4-8 FPS（像素风标准帧率） |

### 6.2 需要的动画集

**通用（1套）**：
- idle（站立呼吸）：4帧循环

**工作状态（每个区域1套，首发6套）**：

| 区域 | 工作动作 | 帧数 |
|------|---------|------|
| 家园 | 整理/烹饪 | 4帧 |
| 竞技场 | 挥剑/跑步 | 6帧 |
| 书阁高塔 | 翻书/写字 | 4帧 |
| 灵感工坊 | 绘画/雕刻 | 4帧 |
| 锻造坊 | 打铁/锤击 | 6帧 |
| 里程碑殿堂 | 审视/环顾 | 4帧 |

**全局状态（5套）**：

| 状态 | 动作 | 帧数 |
|------|------|------|
| active | 精神焕发站立 | 4帧 |
| charging | 安静坐着/冥想 | 4帧 |
| resting | 安睡 | 2帧 |
| traveling | 背包远眺 | 2帧 |
| returning | 走路回来 | 4帧 |

### 6.3 角色制作方式

- **优先从素材包中选取**匹配风格的角色sprite
- 如果素材包角色不合适：以素材包风格为参考，用Aseprite手绘
- 工作动作sprite可基于idle sprite修改（改变手臂/头部姿态）
- **不推荐用AI生成角色sprite**——角色需要帧间一致性，AI很难做到

## 七、装饰物品

### 7.1 装饰规格

| 参数 | 值 |
|------|-----|
| 尺寸 | 16×16 到 32×32（小型装饰物） |
| 风格 | 等距像素，与场景一致 |
| 背景 | 透明 |

### 7.2 每个区域的装饰主题

| 区域 | 装饰物品类型 | 预计数量 |
|------|------------|---------|
| 家园 | 花盆/窗帘/壁炉/地毯/猫咪 | 6-8个 |
| 竞技场 | 奖杯/旗帜/盾牌/训练桩 | 5-6个 |
| 书阁高塔 | 书架/地球仪/卷轴架/水晶球 | 6-8个 |
| 灵感工坊 | 画架/乐器/雕塑/颜料罐 | 6-8个 |
| 锻造坊 | 铁砧/齿轮/蒸汽管/工具架 | 5-6个 |
| 里程碑殿堂 | 火把/柱子/挂毯/宝箱 | 5-6个 |

### 7.3 装饰制作方式

- AI生成为主（独立小物件，AI擅长）
- 统一调色板后纳入
- 每个装饰需要生成在透明背景上

## 八、徽章图标

### 8.1 徽章规格

| 参数 | 值 |
|------|-----|
| 尺寸 | 32×32px 或 48×48px |
| 风格 | 像素风图标，可以比场景更精细一些 |
| 状态 | 已解锁（彩色）+ 未解锁（灰色剪影） |

### 8.2 制作方式

- AI生成基础图标 + Aseprite微调
- 每个徽章需要两个版本：彩色（已解锁）和灰色（未解锁）
- 灰色版本可程序化处理（desaturate + 降低对比度）

## 九、制作优先级与排期

### 9.1 Phase 1：开发阶段（占位图）

**目标**：不阻塞功能开发

| 素材 | 占位方案 |
|------|---------|
| 世界地图建筑 | 彩色方块 + 区域名称文字 |
| 内部场景 | 纯色渐变背景 |
| 角色 | emoji或简单图形 |
| 装饰 | 无（留空位） |
| 徽章 | emoji图标 |

### 9.2 Phase 2：核心素材

**目标**：产品可展示

按优先级排序：
1. 选定素材包，提取调色板
2. 世界地图6个默认区域的建筑（6×6=36张）—— 素材包改
3. 角色idle + 5种全局状态（6套）—— 素材包选/手绘
4. 地图装饰元素和背景（~16张）—— 素材包提取
5. 6个区域的内部场景（6×6=36张）—— AI生成+微调
6. 角色6种工作动作（6套）—— 手绘

### 9.3 Phase 3：完善素材

7. 装饰物品（~40张）—— AI生成
8. 徽章图标（~28张）—— AI生成+微调
9. 空状态插画（3-5张）—— AI生成
10. 碑石6级变化（6张）—— AI生成+微调
11. 里程碑殿堂建筑+内部场景（12张）—— 混合

### 9.4 Phase 4：扩展素材

12. 6个自定义模板区域的全部素材（~114张）—— 混合

## 十一、质量标准

### 11.1 每张素材的验收标准

- [ ] 使用项目统一调色板（无调色板外颜色）
- [ ] 像素边缘锐利，无抗锯齿
- [ ] 等距透视角度正确（2:1）
- [ ] 光照来自左上方
- [ ] 透明背景（PNG格式）
- [ ] 尺寸符合规格
- [ ] 命名符合文件组织规范

### 11.2 整体一致性验收

- [ ] 所有外部建筑放在同一张地图上视觉和谐
- [ ] 角色放在任何场景中都不违和
- [ ] 同一区域6级变化有明确的渐进感
- [ ] 装饰物品与所属区域场景风格匹配
