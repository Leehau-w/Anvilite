## 三、XP计算公式修订

### 3.1 连续活跃加成：线性 → 阶梯式

**替换原 PRD 3.2 节**

旧规则：`+3%/天，上限50%`

新规则：**里程碑阶梯制**

| 连续天数 | 加成 |
|---------|------|
| 1-2 天 | +0% |
| 3-6 天 | +10% |
| 7-13 天 | +20% |
| 14-29 天 | +30% |
| 30 天及以上 | +50%（上限） |

中断处理不变：天数归零，加成归零，不惩罚。

**代码参考**：
```typescript
function getStreakBonus(streakDays: number): number {
  if (streakDays >= 30) return 0.50;
  if (streakDays >= 14) return 0.30;
  if (streakDays >= 7) return 0.20;
  if (streakDays >= 3) return 0.10;
  return 0;
}
```

### 3.2 计算公式：修复取整漏洞

**替换原 PRD 3.4 节**

旧公式（逐项 floor 取整）：
```
最终XP = 基础XP + floor(基础×0.20) + floor(基础×streak%) + floor(基础×0.50)
```

新公式（先汇总倍率，最后统一 round）：
```
最终XP = round( 基础XP × 频率折算系数 × (1 + 按时加成% + 连续加成% + 高难加成%) )
最终矿石 = 最终XP（始终1:1）

其中：
  按时加成% = 0.20（满足条件时，否则0）
  高难加成% = 0.50（难度≥4星时，否则0）
  连续加成% = 0 / 0.10 / 0.20 / 0.30 / 0.50（根据阶梯）
  频率折算系数 = 默认1.0（习惯类见第四章）
```

**修改理由**：旧公式中低难度任务（基础1-3）的单项 floor 经常直接截为0，导致加成形同虚设。新公式先汇总倍率再统一 round，确保中低难度也能享受加成。

**代码参考**：
```typescript
function calculateXP(
  baseDifficulty: number,     // 1/2/3/5/8
  isOnTime: boolean,
  streakDays: number,
  isHardTask: boolean,        // difficulty >= 4
  frequencyCoeff: number = 1.0
): number {
  const onTimeBonus = isOnTime ? 0.20 : 0;
  const streakBonus = getStreakBonus(streakDays);
  const hardBonus = isHardTask ? 0.50 : 0;
  
  const totalMultiplier = 1 + onTimeBonus + streakBonus + hardBonus;
  const finalXP = Math.round(baseDifficulty * frequencyCoeff * totalMultiplier);
  
  return Math.max(finalXP, 1); // 最低保证1XP
}
```

### 3.3 计算示例对比

| 场景 | 基础 | 旧公式结果 | 新公式结果 |
|------|------|-----------|-----------|
| ★任务，按时+连续7天 | 1 | 1+0+0+0=1 | round(1×1×1.4)=1 |
| ★★任务，按时+连续7天 | 2 | 2+0+0+0=2 | round(2×1×1.4)=3 ← 有效提升 |
| ★★★任务，按时+连续14天+高难 | 3 | 3+0+0+1=4 | round(3×1×2.0)=6 ← 显著提升 |
| ★★★★★任务，全加成连续30天 | 8 | 8+1+4+4=17 | round(8×1×2.2)=18 |

## 四、习惯系统数值平衡

### 4.1 频率折算系数

**替换原 PRD 6.1 节**

为平衡高频习惯带来的数值膨胀，引入**频率折算系数**：

| 频率类型 | 折算系数 | 适用 repeatType |
|---------|---------|----------------|

**示例**：难度★★（基础2）的每日习惯，完成后基础计算值 = 2 × 0.5 = 1，后续再叠加各项百分比加成后 round() 取整。

**替换原 PRD 6.2 节**

## 五、习惯模型拓展：灵活次数 + 每月固定日期

### 5.1 数据模型变更

在 `Habit` 接口中**新增**以下字段：

```typescript
interface Habit {
  // ... 原有字段保持不变 ...
  
  // 新增：灵活次数支持
  targetCount: number;          // 周期内目标完成次数（默认 1）
  currentCycleCount: number;    // 当前周期内已完成次数（周期重置时清零）

  // 新增：每月固定日期支持
  monthlyDays: number[] | null; // 例：[1, 15, -1] → 1号、15号和月末最后一天
  
  // 修改：repeatType 新增一个值
  repeatType: 'daily' | 'weekdays' | 'weekly' | 'biweekly' 
            | 'monthly' | 'custom' | 'monthly_fixed';
  // 'monthly_fixed' 区分"每月灵活X次"和"每月指定几号"
}
```

### 5.2 灵活次数业务逻辑

**适用范围**：daily, weekdays, weekly, biweekly, monthly, custom 均可设置 `targetCount > 1`

**打卡流程**：

```
用户点击「完成」
  → currentCycleCount + 1
  → if currentCycleCount < targetCount:
      状态仍为 active
      播放轻量级局部反馈（进度环填充+微小粒子）
      不增加 consecutiveCount
      不结算XP/矿石
  → if currentCycleCount === targetCount:
      状态变为 completed_today
      触发完整反馈（卡片飞出+XP飘字+矿石飘字）
      consecutiveCount + 1
      XP/矿石结算（基础由难度决定，不乘以次数）
```

**防爆肝机制**：
- 同一周期内达到目标次数后按钮置灰，不可再点击
- XP 由难度星级决定，**不受次数乘数影响**
- 即：难度3星的习惯，要求每天做5次，完成后获得 3星的基础XP（即3），不是 3×5=15

### 5.3 每月固定日期业务逻辑

当 `repeatType === 'monthly_fixed'` 时：

**触发规则**：
- 系统读取 `monthlyDays` 数组
- 当天日期在数组内 → 习惯出现在仪表盘"今日待办"
- 当天日期不在数组内 → 习惯不显示，不判定为错过

**月末边界处理**：
- 用户选择31号，但当月无31号 → 自动降级到该月最后一天
- 特殊值 `-1` 代表"每月最后一天"
- 2月28/29日自动适配闰年

**连续性计算**：
- 只有在指定日期打卡才算有效连击
- 非指定日期跳过：不扣容错、不断连、不判定为错过

对于 `targetCount > 1` 的习惯，完成按钮变为**环形进度按钮**：

```
[完成 3/5] 每天练琴5次   🔥12连续  [跳过]
```

- 点击时进度环顺时针填充一部分
- 3/5 → 4/5 → 5/5 时整个按钮变为 ✓ 并触发飞出动画
- 数字使用数字字体规则

## 六、满级长线玩法：淬火重铸系统

### 6.1 触发条件

角色等级达到 **Lv.51**（不朽/传奇称号段）时解锁。

### 6.2 触发交互

弹出特殊的「抉择时刻」模态框：

```
┌──────────────────────────────────┐
│                                  │
│         ⚒️ 抉择时刻               │
│                                  │
│  你已达到不朽之境。               │
│  是否愿意将一切熔于炉火，          │
│  以淬火重铸的方式开启新的篇章？     │
│                                  │
│  重铸后：                         │
│  · 等级重置为 Lv.1               │
│  · 矿石、装饰、主题、繁荣度全保留  │
│  · 获得转生星标 🌟 和专属视觉特权  │
│                                  │
│  [维持现状]          [淬火重铸 ⚒️]  │
│                                  │
└──────────────────────────────────┘
```

- 选择「维持现状」：关闭弹窗，该入口常驻于里程碑殿堂的称号柜旁
- 选择「淬火重铸」：执行重铸逻辑

### 6.3 重铸规则

执行顺序：

```
1. 等级重置：level = 1, currentXP = 0
   （totalXP 历史累计保留，用于统计展示）

2. 资产100%保留：
   - ore（矿石余额）不变
   - totalOreEarned 不变
   - 已购买装饰品全部保留
   - 已解锁主题全部保留
   - 所有区域繁荣度全部保留

3. 转生标识：prestigeLevel += 1

4. 记录 GrowthEvent：type = 'prestige'

5. 徽章颁发：点亮【起源之火】徽章
```

### 6.4 数据模型变更

在 `Character` 接口新增：

```typescript
interface Character {
  // ... 原有字段 ...
  prestigeLevel: number;  // 默认 0，每次重铸 +1
}
```

## 七、反馈动画补充：连击阶梯跃升动画

### 7.1 触发条件

用户当天的完成行为导致全局或习惯连续天数**跨越阶梯阈值**（3、7、14、30天）。

### 7.2 播放时机

在标准的「XP/矿石飘字淡出」后（约1200ms处），如果不与「升级庆祝」冲突则播放。升级庆祝优先级更高——如果同时触发，只播放升级庆祝。

### 7.3 各阶梯参数

**3天阶梯**（轻量提示）：
```
内容: "+10% 加成解锁"
字号: Small (12px)
颜色: text-dim
动画: 向上微飘 20px + 淡出
持续: 1000ms
粒子: 无
```

**7天阶梯**（中等反馈）：
```
内容: "🔥 连击燃起！+20% 经验加成"
字号: H3 (16px)
颜色: accent
动画: 弹性弹出 scale(0.8→1.1→1.0) + 向上飘
持续: 1500ms
粒子: 5-10个，从文字中心向外扩散，accent色
```

**14天阶梯**（中等反馈）：
```
内容: "🔥 连击燃起！+30% 经验加成"
字号: H3 (16px)
颜色: accent
动画: 同7天
持续: 1500ms
粒子: 5-10个
```

**30天阶梯**（隆重庆祝）：
```
内容: "🔥 极意连击！全加成解锁"
字号: H2 (20px)
颜色: accent，闪烁效果（opacity 1→0.6→1 循环2次）
动画: 类似升级庆祝——半透明遮罩 + 弹性弹出 + 大量粒子
粒子: 20-30个，火焰色系（accent + xp混合），中心扩散
持续: 2000ms
可跳过: 点击/ESC
```
