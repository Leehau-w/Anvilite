# Anvilite · 锻石 — 时间线模块规格文档

> **版本**: v1.0
> **日期**: 2026-03-28
> **状态**: 已锁定

---

## 一、模块定位

时间线是用户成长历程的"人生存档"。全量记录所有事件（完成、升级、徽章、里程碑、习惯等），按时间顺序展示，支持回顾和反思。是Anvilite「完整记录成长轨迹」设计哲学的核心载体。

---

## 二、整体形态

### 2.1 布局

**纵向时间轴**，从上到下，最新事件在最上方。

左侧有一条细竖线作为时间轴线条（accent色 20%透明度，宽度2px），事件卡片在线条右侧展开。

### 2.2 三级分组结构

```
▾ 2026年3月                    ← 月分组标题
  ▾ 第13周 (3.25 - 3.31)       ← 周分组标题
    ▸ 3月28日 周五               ← 天分组标题（默认折叠）
      完成5个任务，获得 23 XP     ← 折叠时的摘要
    
    ▾ 3月27日 周四               ← 展开状态
      ⭐ [里程碑] 日语N3通过！     ← 里程碑卡片（大）
      ✓ 完成：日语模拟题 +5 XP    ← 普通事件（小）
      ✓ 完成：跑步5km +3 XP     
      🏆 获得徽章：锤炼 Lv.15     
      ⬆ 升级：Lv.14 → Lv.15     
      ● 习惯完成：背单词 🔥30     

  ▾ 第12周 (3.18 - 3.24)
    ▸ 3月24日 ...
    ▸ 3月23日 ...
```

### 2.3 分组标题样式

**月标题**：
- H2字号(20px)，fontWeight 700，text色
- 粘性定位（滚动时固定在顶部直到下一个月出现）
- 底部边框1px border色

**周标题**：
- H3字号(16px)，fontWeight 600，dim色
- 显示：第X周 (M.DD - M.DD)

**天标题**（可展开/折叠）：
- Body字号(14px)，fontWeight 600，text色
- 格式：`M月DD日 星期X`
- 左侧有展开/折叠箭头 ▸/▾
- **默认折叠**，点击展开查看当天所有事件
- 有里程碑的天：天标题高亮（accent-10%背景），里程碑图标显示在标题右侧

**折叠摘要**（天标题右侧或下方）：
```
完成5个任务，获得 23 XP
```
- Small字号(12px)，dim色
- 数字使用数字字体规则
- 如果当天有里程碑：摘要前显示 ⭐ 标记

## 三、事件卡片

### 3.1 两种卡片尺寸

**里程碑卡片（大卡片）**：
- 用于：用户自定义里程碑、系统里程碑、被标记为里程碑的事件
- 背景：accent-8% + 边框 accent-20%
- padding: 16px
- 圆角: lg(12px)
- 左侧有 ⭐ 图标（accent色，较大18px）
- 标题：H3字号(16px)，fontWeight 700，accent色
- 用户感想（如有）：Body字号(14px)，text色，斜体
- 时间：Small字号(12px)，dim色
- 所在的天标题和周标题也加accent-10%高亮背景

**普通事件卡片（小卡片）**：
- 用于：任务完成、习惯完成/跳过/错过、升级、徽章获得、区域升级等
- 背景：transparent（或交替色hover色）
- padding: 8px 12px
- 无圆角无边框（轻量化，不喧宾夺主）
- 左侧有类型图标（Small字号）
- 内容一行展示
- 时间：Tiny字号(10px)，dim色，右对齐

### 3.2 各事件类型的卡片内容

**任务完成 (task_complete)**
```
✓ 完成：[任务标题]                     +5 XP  ⛏+5    14:32
```
- 图标：✓ success色
- 标题：text色
- XP和矿石：数字字体，success色
- 时间：dim色

**习惯完成 (habit_complete)**
```
● 习惯完成：[习惯名]  🔥30连续          +4 XP  ⛏+4    08:15
```
- 图标：● accent色
- 连续次数：xp色
- XP和矿石：success色

**习惯跳过 (habit_skip)**
```
⏭ 习惯跳过：[习惯名]  连续 30→27                      09:20
```
- 图标：⏭ dim色
- 连续变化：dim色

**习惯错过 (habit_miss)**
```
○ 习惯错过：[习惯名]  容错抵消/连续 30→15              23:59
```
- 图标：○ dim色
- 如果消耗容错：显示「容错抵消」dim色
- 如果连续减半：显示变化，dim色

**升级 (level_up)**
```
⬆ 升级：Lv.14 → Lv.15  锤炼                          14:32
```
- 图标：⬆ accent色
- 等级变化：accent色，fontWeight 600
- 如果跨越称号段：额外显示新称号名

**徽章获得 (badge_earned)**
```
🏆 获得徽章：[徽章名]                                  14:32
```
- 图标：🏆 xp色
- 徽章名：fontWeight 600

**区域繁荣升级 (area_level_up)**
```
🏘 [区域名] 升级为 丰饶 ★★★☆☆☆                        14:32
```
- 图标：🏘 accent色
- 繁荣等级名：fontWeight 600

**用户自定义里程碑 (custom_milestone)** — 使用大卡片
```
⭐ 日语N3考试通过！
   "三个月的努力终于有了回报"                           2026-03-15
```

**系统里程碑 (milestone)** — 使用大卡片
```
⭐ 连续活跃30天
   你已经坚持了整整一个月！                             2026-03-10
```

### 3.3 被标记为里程碑的普通事件

当用户将普通事件标记为里程碑后：
- 该事件从小卡片变为大卡片样式
- 保留原有内容，外层包裹里程碑样式（accent背景+边框）
- 显示 ⭐ 图标
- 可追加填写感想
- 所在天/周标题高亮

## 四、筛选功能

### 4.1 筛选栏位置

时间线顶部，月标题上方，固定不随滚动。

### 4.2 筛选项

**按事件类型**（多选标签）：
```
[全部] [任务完成] [习惯] [升级] [徽章] [里程碑] [区域升级]
```
- 默认选中「全部」
- 可同时选择多个类型
- 标签样式与分类Tab一致

**按区域**（下拉选择）：
```
区域: [全部 ▼]
```
- 下拉列表包含所有已激活区域
- 选择后只显示该区域相关的事件
- 升级、徽章等全局事件在任何区域筛选下都显示

**按时间范围**（按钮组）：
```
[本周] [本月] [全部]
```
- 默认选中「全部」

**搜索框**：
```
🔍 搜索事件...
```
- 按关键词搜索事件标题/描述
- 实时过滤，输入即搜索
- 搜索范围：任务标题、习惯名称、里程碑标题和感想、徽章名称

### 4.3 筛选叠加

所有筛选可叠加使用。例如：类型=「任务完成」+ 区域=「书阁高塔」+ 时间=「本月」→ 只显示本月书阁高塔区域的任务完成记录。

## 五、里程碑标记交互

### 5.1 悬停标记

- 鼠标悬停在任何普通事件卡片上时，右侧出现 ⭐ 按钮（dim色，淡化）
- 点击 ⭐ 按钮：弹出小面板，可选填感想
  ```
  ┌─────────────────────┐
  │ 标记为里程碑          │
  │ 感想（选填）:         │
  │ ___________________  │
  │ [取消]    [确认 ⭐]   │
  └─────────────────────┘
  ```
- 确认后：事件变为大卡片样式，同步到里程碑殿堂碑石
- 已标记的事件：⭐ 按钮变为实心accent色，点击可取消标记

### 5.2 顶部创建里程碑

- 时间线右上角有「+ 创建里程碑」按钮（Secondary按钮 sm尺寸）
- 点击后弹出创建面板（与里程碑殿堂的「刻碑」面板一致）：
  ```
  ┌─────────────────────┐
  │ 刻下里程碑            │
  │ 标题 ____________    │ ← 必填
  │ 日期 [2026-03-28]   │ ← 默认今天
  │ 感想 ____________    │ ← 选填
  │      ____________   │
  │ [取消]    [刻碑 ✓]   │
  └─────────────────────┘
  ```
- 创建后自动插入到对应日期的位置，同步到里程碑殿堂

## 六、时间轴视觉

### 6.1 轴线

- 位置：内容区域左侧，距左边24px
- 样式：2px宽，accent色 20%透明度，纵向贯穿整个时间线
- 每个天分组标题左侧在轴线上有一个小圆点（6px，border色）
- 里程碑所在天的圆点：8px，accent色，发光效果

### 6.2 日期节点

- 普通天：小圆点（6px），border色
- 有里程碑的天：较大圆点（8px），accent色，轻微发光
- 今天：圆点实心accent色 + 脉冲动画

### 6.3 卡片与轴线的关系

- 卡片在轴线右侧展开
- 大卡片（里程碑）左边缘有一条accent色短横线连接到轴线
- 小卡片无连接线（轻量化）

## 七、性能与加载

### 7.1 懒加载

- 初次加载：最近30天的事件
- 向下滚动时：按月加载更早的数据
- 加载指示：底部显示「加载更多...」或加载动画

### 7.2 折叠优化

- 默认折叠所有天（只显示摘要），大幅减少DOM节点
- 展开某天时才渲染该天的事件卡片
- 里程碑所在天的大卡片即使折叠也在摘要中可见（摘要旁显示 ⭐ 标记）

### 7.3 数据量预估

- 假设日均5个事件
- 1年 ≈ 1800个事件
- 3年 ≈ 5400个事件
- 折叠模式下每天只渲染1行摘要，性能不受事件总量影响

## 八、边界情况

### 8.1 新用户（无事件）

- 显示空状态插画 + 「你的故事即将从这里开始」
- 时间轴上只有一个"今天"的节点

### 8.2 筛选无结果

- 显示「没有找到匹配的事件」
- 提示尝试调整筛选条件

### 8.3 大量事件的一天

- 折叠状态下显示摘要（「完成12个任务，获得 58 XP」）
- 展开后所有事件按时间倒序排列
- 无分页，全部展示（一天内事件量通常不超过20个）

### 8.4 撤销任务对时间线的影响

- 撤销完成任务：对应的 task_complete GrowthEvent被删除
- 如果该事件被标记为里程碑：里程碑也一并删除（从碑石和时间线中消失）
- 如果撤销导致降级：level_up事件也被删除

### 8.5 长时间未使用

- 中间空白的天不显示在时间线中（跳过，不显示空行）
- 回归当天显示正常事件

## 九、与其他模块的关系

| 交互 | 说明 |
|------|------|
| 时间线 ← 任务系统 | task_complete/habit_complete等事件来源 |
| 时间线 ← XP系统 | level_up事件来源，XP数据展示 |
| 时间线 ← 徽章系统 | badge_earned事件来源 |
| 时间线 ← 世界地图 | area_level_up事件来源 |
| 时间线 → 里程碑殿堂 | 标记里程碑同步到碑石列表 |
| 时间线 → 里程碑殿堂 | 创建里程碑同步到碑石 |
| 时间线 ← 里程碑殿堂 | 殿堂中创建的里程碑也显示在时间线中 |
