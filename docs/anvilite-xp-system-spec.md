## 二、矿石(Ore)系统 ★v1.1新增

### 2.1 概述

矿石是与XP并行的第二货币，用于购买装饰和解锁主题皮肤。与XP的核心区别：**矿石只增不减**（不因撤销/删除任务而收回），而XP会收回。

### 2.2 获取规则

- 每次完成任务/习惯时，与XP **1:1** 同时获得
- 所有XP加成（按时/连击/高难）同样适用于矿石
- 即：获得的矿石数 = 获得的XP数（始终相等）

### 2.3 消费用途

| 用途 | 说明 |
|------|------|
| 购买区域装饰 | 在区域内部空间的装饰商店中购买，装饰出现在场景中 |
| 解锁主题皮肤 | 晨光白免费默认，其余7套主题用矿石解锁 |

### 2.4 不收回规则

| 操作 | XP | 矿石 |
|------|-----|------|
| 撤销完成任务 | 收回 | **不收回** |
| 删除已完成任务 | 收回 | **不收回** |
| 删除习惯（选择删除模式） | 收回 | **不收回** |

设计理由：矿石代表用户曾经付出的努力的"物质证明"，即使后来撤销了任务，努力本身是真实发生过的。这也避免了"买了装饰后撤销任务导致矿石变负"的问题。

### 2.5 数据结构

```typescript
// 在 Character 接口中
ore: number;              // 当前矿石余额
totalOreEarned: number;   // 历史累计获得矿石（只增不减）
```

### 2.6 显示位置

- 区域内部空间右侧面板（收藏区域标题旁）：`⛏ 42`
- 装饰商店内
- 底部状态栏可选显示

## 三、XP加成系统

加成基于基础XP计算，可叠加。矿石加成与XP加成完全一致。

### 3.1 按时完成加成

- 条件：任务设了截止日期，且在截止日期当天或之前完成
- 加成：基础XP × **+20%**
- 无截止日期的任务不触发此加成

### 3.2 连续活跃加成

- 条件：连续多天完成至少1个任务
- 加成：基础XP × **+3%/天**，上限 **+50%**
- 连续17天时达到上限
- 中断后天数归零，加成归零（不惩罚，只是失去加成）

### 3.3 高难度额外奖励

- 条件：任务难度为★★★★或★★★★★
- 加成：基础XP × **+50%**
- ★★★及以下不触发

### 3.4 计算公式

```
最终XP = 基础XP + 按时加成 + 连续加成 + 高难加成
最终矿石 = 最终XP（始终1:1）

其中：
  按时加成 = floor(基础XP × 0.20)
  连续加成 = floor(基础XP × min(连续天数 × 0.03, 0.50))
  高难加成 = floor(基础XP × 0.50)

所有加成独立计算后相加（基于基础XP，非复利）
```

### 3.5 计算示例

| 场景 | 难度 | 基础 | 按时 | 连续(10天) | 高难 | 最终XP | 最终矿石 |
|------|------|------|------|-----------|------|--------|---------|
| 整理桌面 | ★ | 1 | - | +0 | - | 1 | 1 |
| 背单词(无加成) | ★★★ | 3 | - | - | - | 3 | 3 |
| 写报告(全加成,连续10天) | ★★★★★ | 8 | +1 | +2 | +4 | 15 | 15 |
| 通过考试(全加成,连续30天) | ★★★★★ | 8 | +1 | +4 | +4 | 17 | 17 |

注：基础XP很小，floor取整后低难度任务的加成经常为0。这是有意设计——加成主要奖励高难度+长期坚持的行为。

## 四、等级系统

### 4.1 升级公式

```
升级所需XP = round(5 × ln(level + 1) × level)
```

### 4.2 升级节奏表

| 等级 | 所需XP | 约需天数(日均21XP) |
|------|--------|-----------------|
| Lv.1→2 | 3 | 1天 |
| Lv.2→3 | 11 | 1天 |
| Lv.3→4 | 21 | 1天 |
| Lv.5→6 | 45 | 2天 |
| Lv.10→11 | 120 | 6天 |
| Lv.15→16 | 208 | 10天 |
| Lv.20→21 | 304 | 14天 |
| Lv.25→26 | 408 | 19天 |
| Lv.30→31 | 516 | 25天 |
| Lv.40→41 | 743 | 35天 |
| Lv.50→51 | 982 | ~47天 |

### 4.3 节奏设计约束

- **第一年**：每2周至少升1级 ✓
- **后续**：每月至少升1级 ✓
- **等级不封顶**

### 4.4 关键里程碑

| 时间节点 | 预计等级 |
|---------|---------|
| 第1周 | ~Lv.4-5 |
| 第1月 | ~Lv.9-10 |

| 第6月 | ~Lv.20-22 |

## 六、习惯XP与矿石

### 6.1 基础计算

与一次性任务相同的难度-XP/矿石对应表（1/2/3/5/8），1:1同时获得。

### 6.2 连续完成加成

```
连续完成加成 = floor(基础XP × min(连续次数 × 0.03, 0.50))
```

- 独立于全局连续活跃天数（两套计数互不影响）
- 习惯完成也计入当日"活跃"（影响全局连续活跃天数）
- 矿石加成与XP加成完全一致

## 七、XP相关的系统行为

### 7.1 XP + 矿石获得时

1. 计算最终XP（基础 + 各项加成）
2. 矿石 = 最终XP（1:1）
3. 增加角色的 currentXP、totalXP、ore、totalOreEarned
4. 检查是否满足升级条件
5. 如果升级：currentXP -= 当前等级所需XP，level += 1
6. 记录 GrowthEvent（含 xpGained 和 oreGained）
7. 触发反馈动画

### 7.2 XP收回时（撤销/删除已完成任务）

1. 从 totalXP 中减去对应XP
2. 重新计算 currentXP 和 level（等级和XP必须一致，允许降级）
3. 删除对应的 GrowthEvent
4. **矿石不收回，不做任何调整**
5. 降级时不做特殊动画（静默降级）

### 7.3 连续活跃天数

- 每天完成至少1个任务/习惯算"活跃"
- 连续天数存储在 settings.streakDays
- 规则：昨天活跃→+1，今天已计→不变，更早→重置为1
- 中断不做任何负面提示
