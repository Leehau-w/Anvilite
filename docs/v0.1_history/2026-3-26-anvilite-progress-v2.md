## 二、我们要解决的问题

**核心问题**：人在执行反馈周期长的任务时（学外语、完成大项目、培养兴趣爱好）动力不足。

**市面产品为什么没解决好这个问题**（基于竞品调研的6个根因）：

1. **游戏化不等于创造动力，只能放大已有动力** — 积分徽章无法让人在意一个他们本不在乎的事
2. **外在奖励侵蚀内在动机** — 为了XP做任务而非为了事情本身，一旦奖励感减弱动力更低
3. **惩罚机制与心理安全冲突** — 扣HP、角色死亡在用户低谷时加剧焦虑，形成恶性循环
4. **打卡和任务管理有根本区别** — 没有产品把「可靠的任务规划」和「有成就感的成长系统」真正融合
5. **成长天花板导致长期流失** — 等级封顶、内容耗尽，最忠实的用户反而最先离开
6. **为了量化而量化，忽略了质** — 用户拆碎任务刷XP、标记没做的事为已完成来保连击

## 三、核心设计哲学（已确认，不可更改）

### 3.1 核心公式

**认真的任务管理 × 角色即镜像 × 温和正向引导 × 完整成长记录**

### 3.2 六大设计原则

| 原则 | 说明 | vs 竞品 |
|------|------|---------|
| 任务管理为本，游戏化为翼 | 先做好任务创建、拆解、规划，游戏元素作为副产品自然出现 | Habitica把游戏放在了任务前面 |
| 角色是镜像，不是游戏壳 | Habitica:「完成任务→游戏角色变强」；Anvilite:「你变强→角色映射出你的成长」 | 竞品的XP只是抽象数字，与现实脱节 |
| 进步可视化 > 奖惩驱动 | 核心反馈是「你这周在英语上投入了8小时」，而非「+50XP」 | 竞品用外在奖励替代内在动力 |
| 温和引导，拒绝惩罚 | 不存在HP损失、角色死亡。中断用多样化温和状态表达 | Habitica的扣HP让状态不好的用户更痛苦 |
| 智能任务拆解 | 帮用户把「学好英语」变成可执行的步骤，每步都有反馈 | Habitica只能打卡，Todoist只帮记录 |
| 完整记录成长轨迹 | 全量记录所有事件，形成可回看的人生存档 | 竞品的完成记录转瞬即逝 |

### 3.3 竞品对位

| 竞品 | 它的问题 | Anvilite的回应 |
|------|----------|---------------|
| Habitica | 游戏重管理弱，惩罚制造焦虑 | 任务管理为地基，游戏是自然长出的反馈层 |
| Todoist | 管理强但毫无成就感和反馈 | 每次完成都有可视化反馈，角色映射真实成长 |
| Finch | 温暖但太轻量，无任务管理 | 保留温和引导，加入认真的任务拆解和规划 |
| Forest | 只解决专注，不解决规划 | 帮用户规划做什么，而非只帮不分心 |

## 四、第一版开发总结与反思

### 4.1 第一版完成情况

使用Claude Code进行AI编程，约2小时完成了第一版原型，包含：

- 四页面布局：仪表盘 / 任务 / 角色 / 时间线
- 侧边栏导航 + 底部状态栏
- 任务CRUD + 分类筛选（学习/工作/兴趣/生活/运动）
- XP计算引擎 + 等级系统（已实现Lv.2 初心者）
- 角色状态系统（已实现「充电中」状态）
- 能力雷达图（两个维度：学习/兴趣）
- 成长时间线（全量记录：任务完成、升级、回归事件）
- 连续活跃天数统计

### 4.2 第一版存在的问题

1. **PRD细节不足** — 很多地方AI自由发挥，导致大量细节不满意
2. **养成感薄弱** — 只是一张角色占位图+徽章贴图，和Habitica没有本质区别
3. **缺乏空间感** — 角色系统只是一个平面信息展示页，没有"世界"的感觉
4. **视觉质感不够** — 整体偏"程序员风"，信息都在但缺乏设计感和层次感
5. **项目名称未更新** — 界面仍显示「LifeForge · 千锤百炼」，需更新为「Anvilite · 锻石」

### 4.3 核心教训

**PRD写到80分，AI会用自己的"审美"填补剩下的20分，而那20分往往是决定产品质感的关键。**

第二版的工作重心：把每一个细节讨论到位，产出一份AI无法"自由发挥"的开发方案。

## 五、第二版产品架构（已确认）

### 5.1 最大的架构变化：世界地图系统

第一版的角色页面被替换为一个**三层空间架构**的世界地图系统：

**第一层：世界地图（主视角）**
- 打开「世界地图」Tab默认看到的画面
- 等距像素风的俯瞰视角小世界
- 分成若干区域，每个区域对应一个技能分类
- 区域繁荣度随任务完成而综合变化：建筑升级 + 环境变化 + 装饰增加

**第二层：区域内部空间**
- 点击区域进入
- 可以看到：角色在场景中的状态 + 该分类下的任务列表 + 成长数据面板 + 装饰/收藏展示
- 执行任务时角色在对应的内部空间中（如学习时在书房里）

**第三层：里程碑殿堂**
- 一个特殊的固定区域
- 展示所有里程碑和徽章成就（非简单2D方形贴图，有空间感但不需要3D）
- 包含一个入口可查看角色属性（等级、能力雷达图、称号体系）

### 5.2 导航结构

变化说明：原来的「角色」Tab → 替换为「世界地图」。角色属性视图移入里程碑殿堂。

### 5.3 世界地图区域定义

| 区域名称 | 对应分类 | 可删除 | 可改名 |
|---------|---------|--------|--------|
| 智慧塔 | 学习 | ✓ | ✓ |
| 工坊 | 工作 | ✓ | ✓ |
| 竞技场 | 运动 | ✓ | ✓ |
| 星光阁 | 兴趣爱好 | ✓ | ✓ |
| 家园 | 生活/日常 | ✓ | ✓ |
| 里程碑殿堂 | 成就展示 | ✗ | ✗ |

- 以上为默认预设方案，用户开箱即用
- 用户可自由增加新区域、删除现有区域（里程碑殿堂除外）、修改区域名称
- 每个区域关联一个任务分类，增删区域同时增删对应的任务分类

### 5.4 区域繁荣度变化系统

随着该分类任务完成越多，区域视觉综合变化：

1. **建筑升级** — 核心建筑从简陋逐步升级为宏伟（如茅屋→石屋→城堡），分多个等级
2. **环境变化** — 周围环境从荒芜变繁荣（荒地→草地→花园→繁华），渐进式
3. **装饰增加** — 细节装饰越来越丰富（旗帜、灯光、植被等）

### 5.5 角色状态系统（不变，温和多样化）

| 状态 | 触发条件 | 视觉表现 | 文案 |
|------|---------|---------|------|
| 活跃 active | 近期持续完成任务 | 精神焕发，活跃光效 | — |
| 充电中 charging | 1-3天未活跃 | 安静阅读/冥想，柔和光晕 | 好好充电，随时准备出发 |
| 休憩 resting | 3-7天未活跃 | 星空下安睡 | 好好休息，蓄力再出发 |
| 远行 traveling | 7天以上未活跃 | 背影望向远方 | 无论多久，这里都等你回来 |
| 回归 returning | 中断后重新活跃 | 带着旅行痕迹回来 | 欢迎回来，冒险者 |

设计原则：5种不同状态避免单一化变成惩罚感。每种状态都用温暖正向的视觉和文案。

### 5.6 成长记录策略

- **全量记录**：所有事件（完成、中断、恢复、里程碑、升级等）全部记录
- **成长叙事**：P0不做，后续使用AI生成个性化叙事
- **成长日志筛选**：P0全部显示，后续使用AI智能筛选关键节点
