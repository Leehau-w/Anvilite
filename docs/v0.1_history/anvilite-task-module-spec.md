# Anvilite · 锻石 — 任务模块功能与逻辑规格文档

> **版本**: v1.1
> **日期**: 2026-03-27
> **状态**: 已锁定
> **变更记录**: v1.1 新增任务计时器、矿石获取、区域名称更新

---

## 一、模块概述

任务模块是 Anvilite 的功能地基。它包含两种独立的事物类型：

| 类型 | 入口 | 本质 | 生命周期 |
|------|------|------|---------|
| 一次性任务 | 任务 Tab | 有明确终点，完成即结束 | 创建 → 进行中(计时) → 完成/删除 |
| 周期性习惯 | 区域内部空间 | 无终点，持续重复 | 创建 → 循环（完成/跳过/错过）→ 暂停/归档/删除 |

两者共享同一套分类体系（对应世界地图区域），但在界面上完全独立。

**默认分类（对应世界地图默认区域）**：
- 家园（生活+日常）
- 竞技场（运动/健身）
- 书阁高塔（学习/阅读）
- 灵感工坊（兴趣/创造）
- 锻造坊（工作/效率）

用户可新增自定义分类（对应世界地图自定义区域），总上限12个。

## 二、一次性任务

### 2.1 数据结构

```typescript
interface Task {
  id: string;                        // UUID v4
  title: string;                     // 必填
  category: string;                  // 选填，默认=上次使用的分类
  difficulty: 1 | 2 | 3 | 4 | 5;    // 选填，默认=3
  priority: 'urgent' | 'high' | 'medium' | 'low';  // 选填，默认='medium'
  dueDate: string | null;            // 选填，默认=明天。可手动设为null（无截止日期）
  description: string;               // 选填，默认=''
  estimatedMinutes: number | null;   // 选填，默认=null
  status: 'todo' | 'doing' | 'done';
  parentId: string | null;           // 父任务ID，null=顶层任务
  childIds: string[];                // 子任务ID列表
  nestingLevel: number;              // 嵌套层级，0=顶层，最大=5（即第6层）
  xpReward: number;                  // 基础XP奖励（由difficulty计算）
  actualMinutes: number;             // 实际投入分钟数（由计时器累计） ★v1.1新增
  timerStartedAt: string | null;     // 当前计时开始时间（进行中时有值） ★v1.1新增
  completedAt: string | null;        // ISO 8601
  deletedAt: string | null;          // 软删除时间（回收站用）
  isHidden: boolean;                 // 是否隐藏（已完成任务的隐藏功能）
  sortOrder: number;                 // 手动排序序号
  createdAt: string;                 // ISO 8601
  updatedAt: string;                 // ISO 8601
}
```

### 2.2 难度体系

| 星级 | 文字描述 | 基础XP | 基础矿石 |
|------|---------|--------|---------|
| ★ | 很简单 | 1 | 1 |
| ★★ | 简单 | 2 | 2 |
| ★★★ | 需要专注 | 3 | 3 |
| ★★★★ | 有挑战 | 5 | 5 |
| ★★★★★ | 硬核 | 8 | 8 |

矿石(Ore)与XP **1:1** 同时获得，加成规则也一致。

### 2.3 优先级体系

| 优先级 | 英文 | 颜色规则 |
|--------|------|---------|
| 紧急 | urgent | #dc2626 红色，跨主题固定 |
| 高 | high | 跟随主题辅助强调色 |
| 中 | medium | 跟随主题 |
| 低 | low | dim色 |

默认值为「中」。

### 2.4 状态与流转

```
创建 → [todo]
         ↓ 用户手动切换（开始计时）
       [doing] ← 计时器自动开始
         ↓ 点击勾选框完成（停止计时）
       [done]
         ↓ 撤销
       [todo] 或 [doing]（恢复为完成前的状态）
```

- `todo` → `doing`：用户手动切换，**同时启动计时器**
- `doing` → `todo`：用户手动切换回来，**停止计时器**，已计时间保留
- `todo` → `done`：直接勾选完成（跳过doing，不计时）
- `doing` → `done`：勾选完成，**停止计时器**
- `done` → `todo/doing`：撤销完成
- 任意状态 → 回收站：删除操作

### 2.5 任务计时器 ★v1.1新增

**开始计时**：任务状态变为 `doing` 的瞬间，记录 `timerStartedAt = now()`

**停止计时**：
- 任务完成时：`actualMinutes += (now - timerStartedAt) / 60`，清空 `timerStartedAt`
- 任务切回待办时：同上
- 应用关闭时：自动暂停，`actualMinutes += (now - timerStartedAt) / 60`，清空 `timerStartedAt`
- 应用重新打开时：如果任务仍为 `doing`，重新记录 `timerStartedAt = now()`

**多任务同时计时**：允许多个任务同时处于 `doing` 状态，各自独立计时

**计时器UI**：进行中的任务卡片上显示实时跳动的计时器
```
[■ 停止] 完成日语N3语法笔记      紧急  今天  ⏱ 00:32:15
```
- 勾选框变为「■ 停止」按钮（点击切回 `todo`）
- ⏱ + 实时时间（HH:MM:SS），使用数字字体规则

**计时数据用途**：
- 任务详情抽屉显示「已进行 32分钟」
- 仪表盘角色迷你卡显示「📚 学习中 · 32分钟」
- 区域内部数据面板：该区域累计投入时长

### 2.6 子任务系统

**层级限制**：最多6层嵌套（顶层为第0层，最深为第5层）

**创建子任务**：在父任务的详情抽屉中添加子任务

**父子关系规则**：
- 子任务继承父任务的分类（category），不可单独设置
- 子任务可独立设置难度、优先级、截止日期等其他字段
- 父任务的子任务进度 = 已完成子任务数 / 总子任务数
- 当所有子任务完成时，弹出提示询问是否同时完成父任务（不自动完成）
- 父任务被删除时，所有子任务一并进入回收站
- 子任务可以独立完成，不要求父任务先完成

**展示规则**：
- 列表中：第一层子任务在父任务下方缩进展示，可展开/折叠
- 第2-5层子任务：点击第一层子任务后进入专属子任务视图（类似文件夹进入）
- 有子任务的任务卡片上显示进度（如 3/5）

## 三、任务创建流程

### 3.1 快速创建

**触发方式**：任务列表顶部有一个常驻输入框

**交互流程**：
1. 用户点击输入框，获得焦点
2. 输入任务标题
3. 按回车键
4. 任务被创建，使用所有默认值：
   - 分类 = 上次使用的分类（首次使用时为第一个分类）
   - 难度 = ★★★ 需要专注
   - 优先级 = 中
   - 截止日期 = 明天
   - 其他字段为空
5. 新任务出现在列表中
6. 输入框清空，保持焦点

**分类选择器**（仪表盘快速创建卡片独有）：
- 输入框下方可展开分类快选按钮
- 预设区域直接显示为按钮，自定义区域归在「其他 ▼」下拉中

**边界情况**：
- 空标题按回车：不创建，输入框轻微抖动提示
- 标题过长：不限制字数，卡片上超出用省略号截断

### 3.2 完整创建

**触发方式**：点击「+ 新建任务」按钮

**交互流程**：
1. 从右侧滑出抽屉面板
2. 面板中按顺序展示所有字段：
   - 标题（输入框，自动获焦）
   - 分类（按钮组选择器：[家园] [竞技场] [书阁高塔] [灵感工坊] [锻造坊] [其他 ▼]，一行放不下自动换行）
   - 难度（5星点击选择，带文字描述）
   - 优先级（4个选项按钮：紧急/高/中/低）
   - 截止日期（日期选择器，默认明天，有「无截止日期」开关）
   - 预估时长（数字输入，单位分钟）
   - 描述/备注（多行文本框）
3. 底部有「创建」和「取消」按钮
4. 点击「创建」或 Ctrl+Enter 保存

## 四、任务完成流程

### 4.1 完成交互

**方式一：勾选框快捷完成**
1. 用户点击任务卡片左侧的勾选框
2. 如果难度为1-3星：直接完成
3. 如果难度为4-5星：弹出确认弹窗「确定完成这个任务吗？+XX XP · +XX ⛏」
4. 完成后：
   a. 如果任务在计时中，停止计时器
   b. 卡片向右飞出屏幕
   c. 屏幕底部出现Toast：「已完成 · 撤销」，5秒后消失
   d. 触发全局XP + 矿石动画
   e. 任务移入已完成列表
   f. 记录GrowthEvent（含xpGained和oreGained）

**方式二：详情中完成**
1. 用户点击任务卡片打开右侧详情抽屉
2. 抽屉中有「完成任务」按钮
3. 同样遵循4-5星二次确认规则

### 4.2 撤销完成

**入口一：Toast即时撤销（5秒内）**
- 点击「撤销」：任务恢复为完成前的状态，XP完全收回，成长记录删除
- **矿石不收回**（矿石只增不减，除了购买消费）

**入口二：已完成列表撤销（永久可用）**
- 确认弹窗：「撤销完成将收回 XX XP（矿石不受影响），确定吗？」
- XP收回，可能导致降级。矿石保留。

### 4.3 撤销的连锁影响

- XP完全收回（当作从未完成过），**允许降级**
- **矿石不收回** ★v1.1变更
- 相关GrowthEvent被删除
- 如果是子任务被撤销：父任务的进度相应回退

## 五、任务列表展示

### 5.1 列表分组

默认分组方式：**按状态分组**

```
[进行中] ← 展开，排在最上面，显示计时器
  - 任务A ⏱ 00:32:15
  - 任务B ⏱ 01:05:40

[待办] ← 展开
  - 任务C
  - 任务D

[已完成] ← 不折叠，独立列表区域，带撤销按钮
  - 任务E  +3 XP [撤销]
  - 任务F  +5 XP [撤销]
```

每个分组内的排序：用户可拖拽手动调整顺序。

### 5.2 筛选功能

**分类筛选**：列表顶部Tab栏
```
[全部] [家园] [竞技场] [书阁高塔] [灵感工坊] [锻造坊] [自定义区域...]
```
- 默认选中「全部」
- Tab数量随用户的世界地图区域动态变化（最多12个）

**其他筛选**：
- 按优先级筛选：紧急/高/中/低（可多选）
- 按状态筛选：待办/进行中
- 按截止日期范围：今天/本周/本月

### 5.3 已完成任务展示

- 已完成任务在列表底部有独立区域，**不折叠**
- 每个已完成任务右侧有「撤销」按钮
- 已完成卡片：降低透明度0.7 + 标题删除线 + 显示「+XX XP」
- 用户可选择「隐藏」某个已完成任务

### 5.4 空状态

- 今天还没完成过任务且无待办：插画 + 「还没有任务，创建第一个吧」+ 创建按钮
- 今天完成过且无待办：插画 + 「今天的任务都完成了！」
- 某分类Tab下无任务：「[分类名]还没有任务」+ 创建按钮

### 5.5 过期任务

- 卡片整体淡红色背景（danger-6%）
- 截止日期文字红色，显示「已过期X天」
- 过期任务在对应状态分组的最上面

## 七、删除与回收站

### 7.1 未完成任务的删除

- 确认：「确定删除这个任务吗？」
- 任务进入回收站（软删除）
- 有子任务的任务被删除时，所有子任务一并进入回收站

### 7.2 已完成任务的删除

- 确认：「删除将收回 XX XP（矿石不受影响），你的等级可能从 Lv.X 降至 Lv.Y。确定吗？」
- XP收回、GrowthEvent删除、允许降级
- **矿石不收回** ★v1.1变更

### 7.3 已完成任务的隐藏

- 从列表消失，数据保留，XP不收回
- 可在设置中查看和恢复

### 7.4 回收站

- 入口：任务Tab的某个位置
- 回收站中的任务可恢复或永久删除
- 自动清除：30天后永久删除
- 永久删除后数据不可恢复

### 7.5 分类删除保护

- 如果该分类下还有任务（包括已完成、回收站中的）：不允许删除，提示先移走或删除任务

## 八、周期性习惯

### 8.1 数据结构

```typescript
interface Habit {
  id: string;
  title: string;                     // 必填
  category: string;                  // 必填，默认=上次使用的分类
  difficulty: 1 | 2 | 3 | 4 | 5;    // 必填，默认=3
  estimatedMinutes: number;          // 必填

  // 周期设置
  repeatType: 'daily' | 'weekdays' | 'weekly' | 'biweekly' | 'monthly' | 'custom';
  customIntervalDays: number | null;
  weeklyDays: number[] | null;       // 如 [1,3,5] 表示周一三五
  weeklyMode: 'strict' | 'flexible' | null;

  // 时间范围
  startDate: string;                 // 默认=今天
  endDate: string | null;            // 可选，到期自动归档
  reminderTime: string | null;       // HH:mm

  description: string;

  // 状态与统计
  status: 'active' | 'completed_today' | 'paused' | 'archived';
  consecutiveCount: number;
  totalCompletions: number;
  toleranceCharges: number;          // 0或1
  toleranceNextAt: number;           // 下次获得容错所需连续次数
  lastCompletedAt: string | null;
  lastDueAt: string | null;

  createdAt: string;
  updatedAt: string;
}
```

### 8.2 入口与位置

- 习惯在**区域内部空间**中展示和管理（主要入口）
- 仪表盘习惯卡片显示未完成的（快捷操作入口）

### 8.3 重复周期选项

每天 / 工作日 / 每周（可指定天数+严格/灵活模式）/ 每两周 / 每月 / 自定义间隔

### 8.4 三种结果及连续性规则

**完成** → 连续+1，获得XP+矿石
**跳过**（主动）→ 连续 -= floor(当前×10%)（最少-1），不消耗容错，不获得XP/矿石
**错过**（到期未操作）→ 有容错则抵消（连续不变），无容错则连续减半。不获得XP/矿石

### 8.5 容错机制

- 首次：连续完成满7次获得1次容错
- 之后：每次容错被消耗后，需连续完成14次获得下一次
- 持有上限：1次，不累积
- 仅用于抵消「错过」，「跳过」不消耗

### 8.6 XP与矿石

- 基础XP/矿石由难度决定（1/2/3/5/8），1:1同时获得
- 连续完成加成：floor(基础 × min(连续次数 × 0.03, 0.50))
- 矿石加成与XP加成完全一致

### 8.7 习惯操作

- **暂停**：不计为错过，不影响连续次数（冻结），可恢复
- **归档**：保留所有历史数据和XP/矿石
- **删除**：用户选择归档（保留）或删除（XP收回，矿石不收回，可能降级）

## 九、成长事件记录

| 事件类型 | 触发时机 | 记录内容 |
|---------|---------|---------|
| task_created | 创建任务 | 标题、分类、难度 |
| task_complete | 完成任务 | 标题、xpGained、oreGained、actualMinutes ★v1.1更新 |
| subtask_breakdown | 拆解子任务 | 父任务标题、子任务数量 |
| habit_complete | 完成习惯 | 标题、xpGained、oreGained、连续次数 |
| habit_skip | 跳过习惯 | 标题、连续次数变化 |
| habit_miss | 错过习惯 | 标题、是否消耗容错、连续次数变化 |
| habit_streak | 习惯连续里程碑 | 标题、连续次数 |

撤销完成时：对应GrowthEvent被删除。
